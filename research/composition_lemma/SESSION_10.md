## 2. Interpretation

### 2.1 D^4 НЕ direct sum of simples

Если D^4 = ⊕ L_i (true semisimple decomposition), each L_i would be minimal ideal. Sum of dims = 264. Each L_i contains specific basis elements.

Наша data: union of all proper ideals (where simple summands должны жить) = 145 dim. Остаётся 119 dim where NO proper ideal exists.

Two interpretations:
(a) **D^4 = L_proper ⊕ L_complement** где L_complement is itself simple (no proper sub-ideals). Then L_complement dim 119 = simple, and L_proper dim 145 = some structure (which itself decomposes further).
(b) **D^4 is NOT semisimple** в чистой form. Has solvable radical or non-direct-sum structure.

For (a): dim 119 = 7 × 17 = простые сомножители. Не очевидно matches classical simple Lie algebras. 

Actually wait — if D^4 = L ⊕ M, then for x ∈ M (non-trivial), the ideal generated by x is **at least** M (since M is invariant under D^4-bracket from inside, but bracket с L gives something in M ∩ L = 0, so actually ideal_x ⊆ M). So if there's a single x ∈ M whose ideal is NOT M but something larger, decomposition isn't pure.

Session 9b showed: x ∈ "complement" → ideal_x = full D^4 (dim 264). For direct-sum case, that would require [L, x] ≠ 0, contradicting (a). So **(a) refuted**.

Conclusion: **D^4 is NOT direct sum** L_proper ⊕ L_complement в pure sense. Structure более сложная.

### 2.2 Likely structure: filtration / radical

If D^4 has filtration:
```
0 ⊆ R ⊆ S ⊆ D^4
```
where R = solvable radical (nilpotent ideal), S/R = semisimple, D^4/S = ?

Possibility: 
- **Solvable radical R of dim ≤ 145**: contains the proper ideals.
- **Semisimple quotient D^4/R**: generators of complement project here.

This is consistent с Theorem 20.1 (D^4 = D^5 perfect) — perfect doesn't preclude having radical, just means D^4 = [D^4, D^4]. Even algebras with nontrivial radical can be perfect (if their derived = themselves).

### 2.3 Cryptanalytic significance

Если D^4 has solvable radical R:
- R = nilpotent ideal — slow-mixing directions в SHA's linear layer.
- Elements in R "thermalize" slower than rest.
- This may correspond к specific bit-positions с persistent structure.

**Bridge с Том III IT-4 mechanism (bit5_max signal)**: the 145-dim subspace L_proper might contain the bit positions giving rise к bit5_max signal. **Worth checking** в Session 11/12.

Если L_proper bits correspond к e.g. specific MSBs или LSBs, we have **structural explanation** for IT-4 phenomena. Это бы first time concrete connection между SHA's Lie structure и observed cryptographic biases.

## 3. Bug analysis: signature comparison

### Что сломано

```python
def basis_signature(ideal):
    sorted_basis = sorted(b.tobytes() for b in ideal.basis)
    return tuple(sorted_basis)
```

Two ideals with **same span** but constructed in different order will have different `ideal.basis` (because GF2Basis stores RREF dependent on insertion order). Sorting bytes doesn't fix this — the bytes themselves differ.

### Correct comparison

Two spans equal iff each basis vector of one is в span of other. Test:
```python
def spans_equal(basis_a, basis_b, n):
    # a ⊆ b
    for v in basis_a:
        b_check = GF2Basis(n)
        for u in basis_b: b_check.add(u.copy())
        d_before = b_check.size()
        b_check.add(v.copy())
        if b_check.size() > d_before: return False
    # b ⊆ a (must be checked separately, не automatic)
    for v in basis_b:
        a_check = GF2Basis(n)
        for u in basis_a: a_check.add(u.copy())
        d_before = a_check.size()
        a_check.add(v.copy())
        if a_check.size() > d_before: return False
    return True
```

Or simpler (since both are bases of some span): just check if dim(a + b) = dim(a) = dim(b).

### Impact на Session 10 results

Bug affects "distinct ideals count" (47 may overcounted). Доес NOT affect:
- Per-ideal dimension (45/55/65/75) ✓
- Union dim (145) ✓
- Conclusion that decomposition is non-trivial ✓

Reliable conclusion: D^4 has structure не reducing к simple direct-sum. Session 11 нужна для extracting true count of distinct ideals + identify radical.
