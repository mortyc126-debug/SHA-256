# Session 29: Fixed points of SHA-256 round (no K, W)

**Дата**: 2026-04-25
**Цель**: characterise the fixed-point variety of the SHA-256 round function with K_t = W_t = 0.

## Reduction to a linear condition

Let R be the bare SHA round (no constants, no message). The condition R(v) = v
for v = (a, b, c, d, e, f, g, h):

From b' = a, c' = b, ..., h' = g: a = b = c = d and e = f = g = h.

Plugging in (so Maj(a, a, a) = a and Ch(e, e, e) = e):

```
T_1 = h + Σ_1(e) + e
T_2 = Σ_0(a) + a
a' = T_1 + T_2 = a   ⟹  Σ_1(e) + e + h + Σ_0(a) = 0      ...(I)
e' = d + T_1 = e     ⟹  a + h + Σ_1(e) = 0                ...(II)
```

From (II): h = a + Σ_1(e). Substituting into (I): a + e + Σ_0(a) = 0, so

$$e = (I + \Sigma_0)(a).$$

The chain e = h forces h = e, i.e. a + Σ_1(e) = (I + Σ_0)(a), giving

$$(\Sigma_1 + \Sigma_1 \Sigma_0 + \Sigma_0)(a) = 0.$$

Define M_fix := Σ_1 + Σ_1 ∘ Σ_0 + Σ_0 ∈ End(F_2^{32}). Fixed points of R are
parametrised by **a ∈ ker(M_fix)**, with then e = (I + Σ_0)(a) determined.

## Theorem 29.1 (Trivial fixed-point variety)

**Theorem 29.1.** The bare round R has **exactly one fixed point**, namely the
zero state.

**Proof.** Write Σ_i = I + N_i, with N_i nilpotent (Theorem 24.1, Session 24).
Then over F_2:

$$M_{\text{fix}} = (I + N_1) \cdot N_0 + (I + N_0) = N_0 + N_1 N_0 + I + N_0 = I + N_1 N_0.$$

The product N_1 N_0 is nilpotent: it is a polynomial in s with no constant term
in the commutative ring F_2[s]/(s^{32}) (both Σ_0 and Σ_1 are polynomial
multiplications in this ring, see Session 18).

Hence M_fix = I + (nilpotent) is unipotent, so 1 is its only eigenvalue and
any v ∈ ker(M_fix) satisfies (I + N_1 N_0)(v) = 0, i.e. (N_1 N_0)(v) = v.
But a nilpotent operator has only the zero eigenvalue, so v = 0.

Therefore ker(M_fix) = {0} and R has only the trivial fixed point. ∎

**Verification** (computational): rank(M_fix) = 32, dim ker(M_fix) = 0 ✓.

## Structural decomposition

This is a striking result: **the algebra of Σ_0, Σ_1 forces M_fix to be
unipotent**, regardless of the specific rotation constants.

The argument generalises: **any** SHA-like round with two rotation-only Σ
operators substituted for Σ_0, Σ_1 has only the trivial fixed point in its
bare round. This is a structural property of the unipotent-rotation framework,
not a coincidence of the (2, 13, 22) and (6, 11, 25) constants.

## Role of K_t

If we add round constants K_t to T_1 and T_2, the fixed-point equation becomes
affine:

$$M_{\text{fix}}(a) = K \quad \text{for some } K = K(a).$$

Since M_fix is invertible (Theorem 29.1), a unique a solves this for every K.
However, in actual SHA-256 the constants K_t **vary per round**, so no single
state can simultaneously be fixed by all 64 round operators. The K_t schedule
acts as a deliberate "fixed-point destroyer": it ensures the per-round
dynamics never settles into a stationary state.

This is consistent with NIST's design choice of using disparate K_t (cube roots
of small primes) — the specific values are less important than the fact that
they vary.

## Connection to Lie algebra (Sessions 18–21)

In the Σ-only Lie algebra L_rot (abelian, generated by N_0, N_1):

- M_fix - I = N_1 N_0 ∈ L_rot · L_rot (product of generators).
- N_1 N_0 is nilpotent, in fact in (L_rot)² ⊆ ker of any L_rot-derivation.
- So M_fix lives in I + (radical of L_rot).

This matches the perfect Lie algebra picture (Session 21): the fixed-point
operator stays inside the unipotent radical, never touching the semisimple
part introduced by SHR / σ.
