# Cohomology Probe — честный отчёт

**Дата**: 2026-04-22
**Цель**: измерить 2-когомологическую обструкцию carry через Jacobian rank, проверить соответствует ли она методичке (589/592, §II.7.7) и даёт ли новую структуру для атаки.

## Краткий вывод

Направление **частично закрыто**. Мы нашли реальный, воспроизводимый инвариант (локальный Jacobian carry имеет стабильный rank-deficit), но:

1. **Главная компонента тривиальна**: MSB последнего `W` слова всегда в kernel'е (арифметика mod 2^n — MSB flip не создаёт переносов).
2. **Остальные компоненты state-dependent**: варьируются от anchor к anchor. Это то, что Wang-cryptanalysis УЖЕ эксплуатирует.
3. **Нет новой структурной информации** сверх того, что уже в методичке.

Инвариант подтверждает методичку, но не даёт новых векторов атаки.

## Что измерили

**Локальный Jacobian carry**: для фиксированного `W_base`, флипаем каждый входной бит `W` и смотрим изменение **всех carry-битов** в раунде. Стэкуем в матрицу `input_bits × total_carry_bits`, считаем rank над GF(2).

**Deficit = input_bits − rank**: сколько входных флипов дают линейно-зависимые carry-изменения.

Это дискретный аналог локальной линеаризации + когомологической обструкции (ker(d¹)).

## Результаты

### 1. Deficit зависит от n, НЕ зависит от R (закон сохранения)

| n | mean deficit (R=1..6) |
|---|---|
| 4 | 1.38 |
| 8 | 1.95 |
| 12 | 2.38 |
| 16 | 2.73 |
| 20 | 3.65 |
| 24 | 4.59 |
| 28 | 4.01 |
| 32 | **4.84** |

Для каждого **n**, deficit ≈ const при R=1, 2, 4, 6. Пример n=16:
- R=1: 3.00
- R=2: 2.75
- R=4: 2.55
- R=6: 2.60

Это **первое наблюдение — конкретный закон сохранения**: локальная когомологическая обструкция carry **не аккумулируется через раунды**.

### 2. Соответствие методичке §II.7.7 (589/592)

Методичка говорит carry-rank = 589 из 592 для full SHA-256, deficit = **3**. Наш mini-SHA при n=32 даёт deficit ≈ **4.84**. После вычета тривиальной MSB-компоненты ≈ 3.8. Это в пределах ожидаемого (± разница в точных параметрах mini-SHA vs real SHA-256).

**Подтверждение**: цифра 3 из методички реальна и воспроизводится на mini-SHA.

### 3. Главный вектор kernel'а — тривиален

Анализ `kernel_analysis.py` для n=16, R=4:
- 90% anchors имеют kernel dim = 1
- Этот kernel = {bit 63} = **MSB последнего W слова**

Флип MSB в любом операнде `a + b` **не создаёт новых переносов**: бит суммы в позиции n−1 меняется, но переноса "выше" нет (переполнение отбрасывается mod 2^n).

Это **тривиальная арифметика**, известна криптографам, не даёт атаки.

### 4. Остальные kernel-вектора state-dependent

В ~10% anchors kernel расширяется до dim 2-6. Эти дополнительные направления:
- Варьируются anchor-to-anchor
- Локализованы в ПОСЛЕДНЕМ W слове (bits 5, 11, 13 для n=16)
- Соответствуют carry-cascade termination при специфических операндах

**Это как раз то, что эксплуатирует Wang-cryptanalysis** — подбор дифференциалов где carry ведёт себя детерминированно.

## Связь с методичкой

| Методичка | Наш тест | Статус |
|---|---|---|
| carry-rank = 589/592 (§II.7.7) | deficit ≈ 3-5 на n=32 mini-SHA | ✓ consistent |
| Wang-chain exploits specific diff paths (§II.4) | state-dependent kernel varies | ✓ same phenomenon |
| BTE T5: carry as 1-cocycle (§II.7.6) | R-independent deficit | ✓ consistent with H¹ triviality |

**Не найдено ничего нового**. Всё, что мы измерили, согласуется с известными результатами.

## Что НЕ сработало

- **Надежда на нетривиальный H²**: если бы мы нашли R-зависимый растущий deficit, это была бы новая кумулятивная обструкция. Не нашли — deficit bounded at n-specific constant.
- **Надежда на устойчивый kernel-базис**: если бы мы могли АЛГЕБРАИЧЕСКИ описать kernel (не MSB), это был бы новый инвариант. Не нашли — kernel state-dependent.

## Решение

**Cohomology probe через Jacobian deficit закрыта как источник атаки**. Инвариант реален, но не актионабелен.

Можно ещё попробовать:
- Измерить **global function-level** deficit (rank через миллион random W), не локальный — возможно там другая структура.
- Измерить **2-cocycle condition** напрямую: c(a,b) + c(a+b,c) + c(b,c) + c(a,b+c) для разных троек. Если оно не тождественно 0 — есть H² class.
- **Bracket operation**: вычислить [carry_round_i, carry_round_j] как функции. Если не коммутируют — есть Lie-bracket структура.

Но все три варианта — очень теоретические. Вероятность успеха низкая.
