# IT-5s — SHA-256 Round × Walsh-order chain map

> SHA-256-специфичный эксперимент. Отвечает: как bit5_max информация
> распределяется по Walsh-порядкам state_r при прохождении раундов
> block 1.

---

## 1. Метод

Для каждого раунда r ∈ {4, 8, 12, 16, 20, 24, 32, 48, 64} block 1
SHA-256, вычисляем state_r (посла r раундов compress + добавление IV).
Затем на 256 битах state_r считаем:

- **chain_1(r)** = Σ по 256 битам b: z(state_r[b], bit5_max) · z(state_r[b], state2[bit 10]) / √N
- **chain_2(r)** = Σ по 32 640 парам
- **chain_3(r)** = Σ по 2 763 520 триплетам (C-ядро)

Feature f_in = bit5_max (HW=2, на входе). Target f_out = state2[bit 10]
(финальный выход полного 2-блочного SHA-256).

**RO null**: R=30 реализаций keyed-BLAKE2b 256-битного state (тот же
f_in, f_out). Это null «state_r = random независимо от input».

## 3. Два ключевых наблюдения

### 3.1 Сигнал присутствует на ВСЕХ порядках при малых r

При r=4: chain_1 = 6, chain_2 = 890, chain_3 = 80 000. Отношения
chain_(k+1) / chain_k ≈ 100 — мощное экспоненциальное усиление
с ростом порядка.

Это ожидаемо: state_4 имеет сильные корреляции с input (v20 §106
MI(W; e_1) = 2.5 бит), и каждый дополнительный Walsh-порядок
представляет 256× больше subset-ов, которые могут нести информацию.

### 3.2 Фазовый переход около r=12

| r | chain_3 | изменение |
|---|---|---|
| 4 | −79 696 | baseline |
| 8 | −46 916 | ×0.59 |
| **12** | **+110** | **×−0.0023** (резкий скачок + смена знака) |
| 16 | +727 | ×6.6 |
| 20 | −88 | ×−0.12 |
| 24 | −87 | ×1.0 (стабилизация) |
| 64 | −83 | saturated |

К раунду 12 chain_3 теряет 3 порядка величины и меняет знак. К раунду
20 стабилизируется около RO-state значения −83.5 ± 3.8. Остаётся там
до r=64.

**Содержательное значение**:
- Раунды 1–12: SHA-2 round-function пропускает входную структуру через
  нелинейное смешивание. Walsh-спектр state_r эволюционирует быстро.
- Раунды 12–20: переходная фаза (non-monotonic, r=16 "rebound").
- Раунды 20+: **state_r indistinguishable from random 256-bit state
  по этой метрике**.

## 4. Что означают z_norm вверх 10^4?

`z_norm` при r=4 (например +20999 для chain_3) — это расстояние
SHA-256 state от RO-state в единицах std RO band. Гигантское число
потому что при r=4 state_r **далеко не случаен** — он кодирует input.

RO null "vary state_r" не смешивает это с полезным сигналом.
Правильная интерпретация: z_norm измеряет, насколько НЕслучаен state_r.
Ожидаемо гигантский при r=4 (state почти линеен), малый при r=20+
(state RO-эквивалент).

## 5. Связь с S2 round-decay finding

S2 нашёл: max|z| по 256 output bits при r=4 = 22.67, декремент
exp(−0.25r), равняется RO к r=20.

IT-5s расширяет это:
- **max|z|** (S2 metric) — 1st-order Walsh maximum.
- **chain_1, 2, 3** (IT-5s metrics) — интегральные суммы Walsh-порядков.

Оба говорят одно и то же: **state_r равномерно приближается к
RO-state поведению по мере роста r, с фазовым переходом около r=12–20**.

## 6. Новый факт (не в S2, не в v20)

**Скорость затухания chain_k не зависит от k** в наших измерениях.

При r=4: |chain_3| / |chain_1| ≈ 12 900.
При r=64: |chain_3| / |chain_1| ≈ 83/0.002 = 41 500.

Отношение chain_3 к chain_1 РАСТЁТ с ростом r. Это означает, что
chain_1 быстрее затухает, чем chain_3. Или другими словами: **низкие
Walsh-порядки быстрее становятся RO, высокие держатся дольше**.

Это **квантитативная формулировка** миграции информации в высокие
порядки через round-функцию SHA-256.

## 7. Ограничения и следующие шаги

- RO null "vary state_r" здесь другой чем Q7d null "vary target".
  Q7d null дал z_3 = −3.87 при r=64, здесь z_3 = +0.12. Это не
  противоречие: разные нуль-модели отвечают на разные вопросы.
- Для повторения Q7d-стиля анализа на промежуточных раундах нужен
  дополнительный запуск с варьированием target.
- R=30 недостаточно для Бонферрони на 27 ячейках (α_eff=0.002). 
  При z > 5 (а мы видим |z|>10000 при малых r) это не проблема.
- Фазовый переход r=12 заслуживает более плотного сканирования
  (r=10, 11, 12, 13, 14) для точной локализации.
