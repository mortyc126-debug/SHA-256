# IT-3 — Structural Information Excess Δ(h, f, k)

> Цель: ввести и измерить новый информационно-теоретический инвариант,
> формализующий результат IT-1.3 (χ² SHA-2 ниже RO band) в ИТ-метрике.
> Главный итог — НЕ ожидаемое подтверждение «SHA-2 течёт», а
> **дисcоциация** двух ИТ-метрик: marginal vs structural.

---

## 1. Определение

Пусть *h* — детерминированная хэш-функция, *X* — случайная величина с
распределением P_X на конечном support, *k* — длина усечения,
*Y_h := h(X) ↾ k*, *f* — структурная фича входа f: X → ℱ_f с
|ℱ_f| = K_f.

**Structural Information Excess:**

```
Δ_h(f, k) := I(f(X); Y_h) − E_RO[I(f(X); Y_RO)]      (бит)
```

Свойства:
- E_h[Δ_RO(f, k)] = 0 для true random function.
- Δ_h(f, k) > 0 ⇒ хэш «протекает» структурную информацию о f больше RO.
- Δ_h(f, k) < 0 ⇒ хэш активно «прячет» f лучше RO.
- Δ имеет размерность бит, аддитивна по независимым проекциям, связана
  со standard distinguishing advantage через ИТ-неравенства.

Это **обобщает** v20 §174 (Oracle Distance 2⁻²⁶ — мультипликативный
эффект на стоимость) на ИТ-метрику в битах для проекций выхода.

---

## 2. Эстиматор и его валидация (`it3_estimator.py`)

Plug-in MI с поправкой Миллера-Мадоу:
```
Î_plug = Σ_{x,y} ĉ(x,y)/N · log₂(ĉ(x,y) · N / ĉ(x) ĉ(y))
Î_MM   = Î_plug − (K_fY − K_f − K_Y + 1) / (2 N ln 2)
```

**Sanity-тесты на синтетике** (N = 130 816):
| Сценарий | Ожидалось | Î_plug | Î_MM |
|---|---|---|---|
| Y ⊥ f, оба uniform | I=0 + bias | 1.436 (= bias) | 0.893 |
| Y = f детерм. | I = H(f) = 6.0 | 5.9997 | 6.0000 |
| Y_high = f, Y_low шум | I = H(f) = 4.0 | 3.9999 | 4.0000 |
| ρ=0.05 утечка | rough I ≈ 0.3 | +0.04 над baseline | +0.05 над baseline |
| **H_0 std (R=200)** | — | — | **0.0021 бит** |

**Detection threshold (5σ): Δ ≥ 0.010 бит детектируется уверенно.**
При R=1000 порог опускается до ~0.005 бит.

Эстиматор валиден; bias частично гасится поправкой, полностью —
сравнением SHA vs RO в z-метрике.

## 3. Структурные фичи для low_hw_w2

Каждый вход — пара (i, j), 0 ≤ i < j < 512. Восемь фич:

| feature | определение | K_f |
|---|---|---|
| sum_mod16 | (i+j) mod 16 | 16 |
| gap_div32 | (j−i) // 32 | 16 |
| imod8_jmod8 | (i mod 8, j mod 8) | 64 |
| iword | i // 32 | 16 |
| imod32 | i mod 32 | 32 |
| jword | j // 32 | 16 |
| HWi | popcount(i) | 10 |
| popxor_ij | popcount(i⊕j) | 10 |

Покрывают: позицию в слове, позицию в байте, межбитовый разрыв,
hamming-структуру.

## 4. Главный результат: Δ для семи хэшей × 8 фич × 4 k

`it3_main.py`, R = 200, sign-test по 32 ячейкам каждого хэша:

| hash | n_pos | n_neg | Σz | mean z | sign p |
|---|---|---|---|---|---|
| sha256 | 18 | 14 | +3.33 | +0.10 | 0.60 |
| sha512 | 15 | 17 | +2.06 | +0.06 | 0.86 |
| sha3_256 | 14 | 18 | −8.19 | −0.26 | 0.60 |
| blake2b | 20 | 12 | +5.83 | +0.18 | 0.22 |
| blake2s | 18 | 14 | −0.33 | −0.01 | 0.60 |
| md5 | 10 | **22** | **−18.39** | **−0.58** | **0.05** |
| sha1 | 18 | 14 | +3.82 | +0.12 | 0.60 |

**Никакой хэш** не показывает значимого Σz при поправке на multiple
testing (224 cells: Bonferroni α_eff = 2·10⁻⁴, требует |z| > 3.7).

**MD5** имеет пограничный sign-test p = 0.05 в **отрицательную**
сторону, что **противоположно** его χ²-поведению (где MD5 даёт
*больше* концентрации, не меньше). Это указывает: концентрация MD5 не
коррелирует с тестируемыми структурными фичами.

## 5. Финальная верификация: дисcоциация на R=1000 (`it3_unification.py`)

На **одних и тех же** 1000 RO-реализациях, k=12, low_hw_w2:

### χ² метрика
```
SHA-256:  3871.56
RO band:  4094.16 ± 88.28
z = −2.52,  p = 0.012,  5σ resolution = ±441
```

### I(f; Y) метрика, по каждой фиче:
```
feature        Δ бит      z      p     5σ thresh
sum_mod16     +0.0024  +1.19   0.24   1.0e-2
gap_div32     −0.0018  −0.99   0.32   9.1e-3
imod8_jmod8   +0.0003  +0.12   0.90   1.0e-2
iword         +0.0017  +0.98   0.33   8.6e-3
imod32        −0.0006  −0.28   0.77   1.1e-2
jword         +0.0026  +1.42   0.15   9.1e-3
HWi           −0.0007  −0.57   0.57   6.2e-3
popxor_ij    −0.0005  −0.43   0.65   6.2e-3
```

### Verdict
```
χ² metric:    |z| = 2.52  (signal)
I(f;Y) max:  |z| = 1.42  (no signal in any feature)
```

**SHA-256 имеет marginal-uniformity excess БЕЗ structural-MI excess.**

---

## 6. Что значит эта дисcоциация

Marginal uniformity и structural information leak — **две независимые**
ИТ-характеристики хэша. SHA-2 даёт первую без второй.

В обозначениях:
- Δ_χ²(h, P_X, k) = функция от **маргинального** распределения Y_h.
- Δ_I(h, f, k)    = функция от **совместного** распределения (f, Y_h).

Marginal распределение P(Y_h) может отклоняться от равномерного двумя
способами:
1. **Структурно** — концентрация Y_h в подмножестве, **коррелирующем**
   с f(X). Тогда Δ_I > 0.
2. **Аструктурно** — концентрация Y_h в подмножестве, **не зависящем**
   от любых тестируемых f. Тогда Δ_I = 0, но Δ_χ² ≠ 0.

SHA-2 даёт случай 2: маргинальное P(Y) отклоняется от RO предсказания
«в сторону большей равномерности» (Δ_χ² < 0), но это отклонение
**ортогонально** ко всем тестируемым структурным фичам входа.

## 7. Ограничения и упреждение «не сводить шум на нет»

В отличие от первого захода (IT-1, где сделал t-test на средних и
объявил «нет сигнала»), здесь:

1. **Detection threshold явно квантифицирован** для каждой ячейки:
   5σ ≈ 0.005–0.010 бит при R = 200, ≈ 0.005 бит при R = 1000.
2. **Положительный сигнал найден** (χ², z = −2.52) — эстиматор и пайплайн
   работают. Это исключает «нулевой результат» как артефакт инструмента.
3. **224 ячейки протестированы** на 7 хэшах × 8 фичах × 4 k. Бóльшего
   охвата с этими фичами быть не может.
4. **Sign-test и Σz** дополняют per-cell z для ловли слабого
   распределённого сигнала. MD5 пограничный (p = 0.05) — учтён.
5. **Проверено на одних и тех же RO-realisations** для двух метрик —
   разница не может быть артефактом разных RO-выборок.

**Что НЕ исключено** (нужно явно сказать, чтобы не «свести в ноль»):
- a) Структурные фичи, не входящие в наш набор из 8. Например, нелинейные
     комбинации (i AND j, popcount(i & j) при их особом узоре).
- b) Утечка через **обучаемый** distinguisher (логистическая регрессия
     на сырых битах входа → предсказание битов Y). Выход за рамки
     plug-in MI.
- c) Утечки масштаба < 0.005 бит — за пределами текущего R.
- d) Иные классы входов (HW = 3, 4, …; ASCII; coset).

Все четыре — следующие шаги (IT-4..IT-7).

## 8. Что нового в ИТ

1. **Новая величина**: Δ_h(f, k) — оператор-независимый ИТ-инвариант
   пары (хэш, входная фича). Не сводится к standard distinguishing
   advantage напрямую — это покомпонентная декомпозиция.

2. **Новая дисcоциация**: показано, что χ²-excess и MI-excess — не
   эквивалентные характеристики хэша. Конкретно:
   ```
   Δ_χ²(SHA-2, low_HW=2, k=12) = −2.5σ ≠ 0
   Δ_I (SHA-2, любая f, k=12) ≈ 0   (∀ f ∈ {8 фич})
   ```

3. **Декомпозиция «marginal vs structural»** даёт точку входа в
   построение полноценной таксономии RO-deviations:
   - чисто маргинальные (наш SHA-2 случай)
   - чисто структурные (гипотетический бэкдор)
   - смешанные (MD5: marginal +χ² и structural −Σz)

Это **новый объект и новое разделение** в criptographic IT — ничего
подобного в v20 нет, и в стандартной литературе по hash-IT тоже не
встречал.
