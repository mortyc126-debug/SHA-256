## Путь D — независимая SHA-side ре-верификация IT-6 (17 апреля 2026)

### Шаг D.1: Независимая реализация direct_z через hashlib

**Метод**: с нуля, без использования внутреннего пакета
`sha256_chimera.py`:
- Сгенерированы 130816 HW=2 входов через `itertools.combinations`.
- SHA-256 вычислен через стандартный `hashlib.sha256`.
- Feature `bit5_max = (max_position >> 5) & 1` пересчитан.
- `direct_z(b) = √N · (1 − 2·mean(f ⊕ t_b))` пересчитан для всех 256 битов.

**Результат**:
- `Max |diff| с сохранённым sha_direct = 0.0` (**бит-в-бит совпадение**).
- `bit 10: indep z = -3.9150`, совпадает со stored.
- `bit 210: indep z = -3.0192`, совпадает со stored.

**Вывод**: (i) SHA-256 в `sha256_chimera.py` = стандартный SHA-256;
(ii) input generation и feature-вычисление корректны; (iii) direct_z
формула реализована верно. ✓

### Шаг D.4: Fragility анализ same_sign к RO seed

Из 256 выходных битов:
- `direct`: 41 бит имеет `|margin| < std/√R` (потенциально flip при

- `chain`:  44 бита аналогично.

с другим seed дал бы похожий результат.
