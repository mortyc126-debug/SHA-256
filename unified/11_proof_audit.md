# Путь B — Аудит доказательств / ре-верификация фундамента

> Аналитическая проверка + точечный brute-force для центральных теорем
> Томов I-II. Цель: установить, что базовые константы и структурные
> утверждения, на которые опирается вся методология, корректны.

## Метод

Для каждой теоремы:
- **[CONST]** — численные константы сверяются с SHA-256 IV/K и
  формулами Σ/σ/Ch/Maj.
- **[EXH]** — exhaustive brute-force по полному пространству (для
  компактных параметров).
- **[RAND]** — большая выборка случайных тестов (N ≥ 10 000).
- **[LINALG]** — прямое вычисление ранга матрицы над GF(2).

## Аналитические теоремы

### T_DCH_EXACT [П-22/П-23]

**Формулировка**: `Ch(e⊕δ, f, g) ⊕ Ch(e, f, g) = δ ∧ (f ⊕ g)`.

- **[EXH]** 4-битное: 65536 / 65536 MATCH.
- **[RAND]** 32-битное: 10000 / 10000 MATCH.
- **Вывод**: ✓ строго корректна.

### Σ0/Σ1/σ0/σ1 XOR-линейность

Все 4 функции — сумма (XOR) F₂-линейных отображений (ROTR, SHR),
значит F₂-линейны.

- **[RAND]** Σ1: 10000/10000 `Σ1(a⊕b) = Σ1(a)⊕Σ1(b)`.
- **[RAND]** Σ0/σ0/σ1: 1000/1000 каждый.
- **Вывод**: ✓ строго.

### L4 [§II.1.1]: HW(Σ1(2^k)) = 3 для всех k=0..31

Утверждение: три rotation-destinations {(k-6)%32, (k-11)%32, (k-25)%32}
попарно различны (смещения 5, 19, 14 mod 32 различны).

- **[EXH]** все k=0..31: HW(Σ1(2^k))=3 ✓
- Идентично для Σ0: HW(Σ0(2^k))=3 ∀k ✓
- **Вывод**: ✓ строго.

### T_SCHEDULE_FULL_RANK [П-23, П-201]

**Формулировка**: Матрица XOR-расписания L: GF(2)⁵¹² → GF(2)¹⁵³⁶ имеет
rank = 512 (полный ранг). ker(L) = {0}.

- **[LINALG]** явное построение матрицы 2048×512 (64 выходных слова ×
  32 бита) из базисных векторов → ранг по Гаусс-Жорадану:
  - Полная матрица 2048×512: rank = **512** ✓
  - Output-only 1536×512: rank = **512** ✓
- **Вывод**: ✓ строго.

### T_HW64_INDEPENDENCE

**Формулировка**: HW(SHA-256(W)) ~ Bin(256, 0.5); corr(hw(W), hw(W⊕e)) ≈ 0.

- **[RAND]** N=50000 входов:
  - HW SHA-256 mean=128.01, std=8.00 (идеальное Bin(256, 0.5))
  - hw64 (первые 64 бита): mean=31.99, std=3.99 (идеальное Bin(64, 0.5))
- **[RAND]** Корреляции `corr(hw(W), hw(W⊕bit_e))` для 6 разных e:
  все |ρ| ≤ 0.015 (шум).
- **Вывод**: ✓ эмпирически.

### T_IV_BIT0 [П-21]

**Формулировка**: `C = d_iv + T1_0_base = 0x98c7e2a2`, bit₀(C) = 0,
следовательно `δe1 = 1` для любого W0 при `δW0 = 1`.

- **[CONST]** bit₀(0x98c7e2a2) = 0 ✓
- **[RAND]** N=5000: bit₀(δe1)=1 во всех случаях при δW0=1 ✓
- **Вывод**: ✓ строго.

### T_SUFFICIENT_R1 [П-22]

**Формулировка**: `W0[0..14] ≥ 7518 AND W0[15] = 0 → δe1=0x8000` с P=1.

- **[EXH]** 2¹⁶ значений W0_low: все 25250 попадающих под signature
  → δe1=0x8000. **0 нарушений.**
- **[RAND]** 100 000 случайных W0: 38 513 с signature, все дают
  δe1=0x8000.
- Эмпирическая доля ≈ 0.385, что соответствует (2¹⁵−7518) × 2¹⁵ / 2³¹
  ≈ 0.385 ✓.
- **Вывод**: ✓ строго.

### T_SC_A1 / T_JOINT_SC [П-24]

**Формулировка**: `δa1=0x8000 ⟺ bit15(a1_n)=0` (conditioning на W0[15]=0).

Нашёл **имплицитное условие**: биусловие держится **при W0[15]=0**.
Без этого условия — только в случаях (W0[15], a1_n[15])=(0,0) или
(1,1), т.е. в ~50% случаев, не биусловие.

- **[RAND]** N=100 000:
  - Conditional on W0[15]=0: 50 260 / 50 260 (= 100%) MATCH ✓
  - Без условия: вместо биусловия имеем `δa1=0x8000 ⟺ W0[15]=a1_n[15]`.
- **JOINT_SC**: `W0[15]=0 AND W0[0..14]≥30643`:
  - Доля: 3.26% ≈ 3.2% claim ✓
  - Все 3263 случая дают **одновременно** δe1=0x8000 AND δa1=0x8000 ✓
- **Вывод**: ✓ эмпирически корректна; формулировка в unified
  требует уточнения о W0[15]=0 (неявное условие).

**Рекомендация**: в `unified/02_vol_II/04_wang_chain_p23_p101.md §II.4.3`
явно указать условие W0[15]=0 для биусловия T_SC_A1.

## Теоремы, требующие доп. контекста

### T_CH_INVARIANT [П-966]

**Формулировка**: `Ch[b30, b31] = 0 при carry[63] = 0` (0/1M).

- Моя интерпретация `carry[63]` = carry-out of (T1+T2) на позиции 31
  в финальном раунде 63 **не подтверждает** утверждение (из 5000
  тестов — ~1260 нарушений при carry=0).
- Без оригинального источника кода точное определение `carry[63]`
  неясно; может быть специфический carry-trace в 64-bit
  представлении каскада.
- **Статус**: **требует уточнения формулировки** перед ре-проверкой.
  Скорее всего корректна в своей original-сетке (0/1M за заявленное),
  но публичная формулировка в unified слишком лаконична.

### T_MERSENNE_DECOMPOSITION [§II.7.8]

**Формулировка**: В кольце Z/(2³²−1) сложение carry-свободно,
SHA-256 = Мерсенн-вычисление + Ch/Maj (степень 2) + 592 бинарных
коррекций.

- Формально: Z/(2³²−1) это кольцо; сложение в нём — вычисление
  (a+b) mod (2³²−1) = a+b при a+b < 2³²−1, иначе a+b−(2³²−1).
  Carry propagation заменяется на **одну глобальную коррекцию**
  (вместо цепочки из 32 carry bits).
- Claim «592 бинарных коррекций» это структурное число,
  происходящее от карты carry-bit distribution по 64 раундам.
- Формула carry-rank = 589/592 — эмпирическая наблюдение на
  матрице образов, требует Matrix construction из Vol II
  (отсутствует в репо).
- **Статус**: **claim-level правдоподобен**, но конкретные числа
  (589, 592, 2⁻⁷⁷) требуют ре-вычисления по оригинальным скриптам.

### T_CARRY_ANALYTIC [П-22]

**Формулировка**: `P(δe1 = 2^k) = C[0..k-1]/2^k` (if C[k]=1), или
`(2^k − C[0..k-1])/2^k` (if C[k]=0).

Формула зависит от битов константы CONST_e1 = 0x98c7e2a2.
- Эмпирически проверена для **k=0** (T_IV_BIT0) ✓.
- Для общих k требуется счёт по диапазонам W0, что делается
  скриптом но не повторено здесь.
- **Статус**: ✓ частично (k=0), полная проверка требует отдельного
  скрипта по П-22.
