## 1. Введение и цель

### 1.1 Основной вопрос

Существует ли классическое расширение бита как вычислительного
примитива, аналогичное кубиту, но без квантовой физики? Если да —
сколько таких расширений, как они связаны, и какое из них самое
мощное?

### 1.2 Что значит «расширение бита»

Обычный бит — это элемент множества {0, 1} с алгеброй XOR/AND. За
этим стоит очень специфический выбор:

- значения дискретны
- операции бинарны
- память о состоянии не хранится
- нет ни времени, ни порядка, ни контекста
- нет знака, амплитуды, фазы
- нет корреляций между битами как первичной структуры
- нет ограничений на использование

Каждое «снятие» одного из этих ограничений даёт новый примитив —
математический объект, который можно назвать «бит» в обобщённом
смысле, но который обладает хотя бы одной вычислительной
способностью, которой нет у обычных {0, 1}.

Эти обобщения должны быть **строго классическими**: никакой
квантовой физики, никаких комплексных амплитуд, никаких
вероятностных источников случайности, требующих физического
устройства. Всё на целочисленной арифметике с проверяемыми
инвариантами.

### 1.3 Мотивация

Основная исследовательская рамка пользователя — «чем больше
ленишься, тем больше я здесь буду сидеть»: строгий упор на
реальный прогресс, комбинирование существующей математики вместо
попыток изобретать с нуля. Отсюда следует методология:

- Каждый новый примитив должен **комбинировать** уже известные
  математические структуры (линейная логика, теория узлов,
  модальная логика, лямбда-исчисление и т.д.), а не придумывать
  новое на пустом месте.
- Каждый примитив должен быть **реализован** в конкретном C-файле
  с набором экспериментов и проверяемыми инвариантами.
- Результаты должны быть **измеримыми**, а не декларативными. Если
  заявляем, что X не сводится к Y, должна быть хотя бы одна
  конкретная характеристика, которая это различает.
- **Честность относительно ограничений**: если что-то не
  получается (как R=2 инверсия SHA-256), это надо фиксировать
  как отрицательный результат, а не замалчивать.

### 1.4 Две параллельные ветки программы

В ходе сессии программа пошла по двум направлениям, которые
встретились только в конце:

**Ветка 1 — криптоанализ и математика битов в контексте SHA-256.**
Начали с попытки прорваться в SHA-256 с помощью HDC-памяти и
нейронных сетей из битов. Получили один сильный положительный
результат (1765× ускорение на 1 раунде) и четыре независимых
измерения фазового перехода R=3→R=4, после которого SHA-256
становится криптографически случайным по всем известным нам
метрикам.

**Ветка 2 — таксономия расширений бита.** Сосредоточились на
поиске новых осей расширения, начиная с фазовых битов (знак +
интерференция) и заканчивая 12-осевой классификацией.

В обоих случаях метод одинаковый: брать хорошо изученную
математическую рамку (спектральная теория графов, линейная
логика, persistent homology, braid-группы и т.п.) и аккуратно
переносить её на чисто битовый субстрат.

## 2. Контекст до сессии

До этой сессии в репозитории была накоплена большая ветка работы

## 3. Фаза A — HDC алгебраические варианты

### 3.1 Общая идея

Классический HDC (Kanerva) использует бинарные HDV длины D=2048
с операциями:

- **bind** (`⊗`) = XOR по позициям — коммутативно, инволютивно,
  сохраняет ортогональность
- **bundle** (`⊕`) = majority — ассоциативно, коммутативно,
  накапливает в bundle
- **sim** = 1 − Hamming/D — стандартная метрика

Цель фазы A — перенести классический HDC в новые алгебраические
контексты, заменяя одну или несколько базовых операций на
что-то более богатое.

### 3.2 ternary_hdc.c — полурешётка {−1, 0, +1}

**Новизна**: третий символ «0» = «не знаю / безразлично».
Каждая позиция хранится как 2 бита (plane0, plane1):

- `(0, 0)` = 0 (не определено)
- `(1, 0)` = +1
- `(0, 1)` = −1
- `(1, 1)` = противоречие (отбрасывается)

**Операции**:

- `bind(a, b)`: 0 стирает (0 ⊗ x = 0), иначе произведение знаков
- `bundle(a, b)`: majority, игнорируя 0-позиции
- `sim(a, b)`: только по непустым позициям

**Результаты**:

- IS-A иерархия сохраняется идеально (animal ⊃ dog ⊃ dachshund)
- Capacity на 22% выше бинарного при k=3
- 100% accuracy на 3-class test
- **Образует полурешётку**: есть частичный порядок на HDV через
  «один содержит нули там, где у другого есть значения»

### 3.3 quaternary_hdc.c — билаттис Белнапа

**Новизна**: 4 значения {0, +1, −1, ⊥}, где ⊥ — «противоречие».
Это классический билаттис Белнапа (Belnap 1977), оформленный
как структура из двух частичных порядков:

- truth order: −1 < 0 < +1 (обычная логика)
- information order: 0 < (+1, −1) < ⊥ (сколько мы знаем)

**Ключевая операция**: `merge(a, b)` = info-join, который
автоматически детектирует ⊥ при слиянии противоречащих источников.

**Результаты**:

- 5-source fusion test: детектировано 1691 противоречие
- Парапоследователен: система продолжает работать при наличии ⊥
- merge ассоциативен, коммутативен, идемпотентен
- **Первое обобщение HDC за пределы классической {0, 1} логики**

### 3.4 topological_hdc.c — персистентная гомология

**Новизна**: применяем persistent homology (Carlsson 2009) к
облаку HDV в Hamming-метрике. Считаем $H_0$ через Vietoris-Rips
филтрацию: каждый порог сходства создаёт граф, union-find
отслеживает слияния компонент.

**Инварианты на датасет**:

- max persistence (время жизни самого долгого кластера)
- mean persistence
- Число компонент при каждом пороге
- Betti number $b_0$ как функция сходства

**Результаты**:

- Cluster detection без указания k работает: 3 истинных кластера
  обнаруживаются по трём высоко-персистентным компонентам
- Partial success — нужна настройка порогов
- **Первое применение PH к чистым bit HDV**
- Баг: death=−1 давал persistence=1.0. Исправлен: death=min_sim
  для «выживших» компонент.

### 3.5 information_hdc.c — Шеннон + MDL + LPN

**Новизна**: комбинирует HDC с теорией информации:

- **Shannon entropy** per-position: $H_i = -p\log_2 p - (1-p)\log_2(1-p)$
- **Minimum Description Length** (Rissanen 1978): HDV как индекс
  в словаре + residual битов
- **XOR-decomposition** = LPN hardness

**5 экспериментов**:

1. Entropy: случайные HDV → 99.3% от D, структурированные → 75.6%,
   идентичные → 0%
2. MDL vs dictionary size: 52% экономии при dict=100+
3. XOR-decomposition fails after 1 component — **это LPN**
4. Bundle-decomposition succeeds — top-4 находят всех членов
5. Learning curve

**Ключевое открытие**: один и тот же HDV-субстрат даёт две
противоположные вычислительные роли в зависимости от binding:

- XOR binding → Learning Parity with Noise → **криптографическая
  односторонняя функция**
- Bundle → correlation-addressable → **ассоциативная память**

Композиция дешёва в обоих случаях; декомпозиция крипто-трудна под
XOR и тривиальна под bundle. HDC, таким образом, **унифицирует
криптопримитивы и ассоциативную память в одной алгебре**.

### 3.6 fibonacci_hdc.c — Zeckendorf, золотое сечение

**Новизна**: HDV в языке «Fibonacci-допустимых» строк — бинарных
строк без двух соседних единиц. Теоретический фундамент:

- **Теорема Цекендорфа** (1972): каждое целое уникально
  представимо как сумма неподряд идущих чисел Фибоначчи
- **Плотность**: $p/(1+p)$ где $p = 1/\varphi^2 \approx 0.382$
- **Энтропия**: $\log_2\varphi \approx 0.694$ бит/позиция

**Результаты**:

- Измеренная плотность 0.276 ≈ $p/(1+p)$ — теория подтверждена
- Псевдо-ортогональность сохранена: mean 0.600, std 0.011
- Zeckendorf encoding чисел: 100 = F₁₁ + F₆ + F₄
- **Логарифмическое сохранение метрики**: $|a-b|=1 \Rightarrow$
  Hamming≈2, $|a-b|=10^4 \Rightarrow$ Hamming≈10
- **Почти-линейность**: sim(Z(a+b), Z(a)⊕Z(b)) ≈ 0.997

Даёт HDC теоретико-числовой фундамент: разреженность из золотого
сечения, уникальная каноническая форма, почти-линейная арифметика
с carry-коррекциями.

### 3.7 spectral_hdc.c — Fiedler, Cheeger

**Новизна**: спектральная теория графов (Fiedler 1973, Chung 1997,
Shi & Malik 2000, Cheeger 1970) применяется к HDV-графу с весами
$w_{ij} = \max(0, 2\cdot sim - 1)$. Нормализованный Laplacian
$L = I - D^{-1/2}WD^{-1/2}$, диагонализация через Jacobi (O(N³),
N≤80).

**Пять результатов** (все 4/4 чистые):

1. **Fiedler ratio 30.57×** различает случайные ($\lambda_1=0.617$)
   и кластеризованные ($\lambda_1=0.020$) HDV-графы
2. **Eigengap детектирует k точно** для k=2,3,4,5
3. **Spectral embedding $\mathbb{R}^{2048} \to \mathbb{R}^3$**:
   3 кластера образуют равносторонний треугольник (inter-centroid
   distances все = 0.36515)
4. **Cheeger's inequality** проверено: Fiedler sign-cut даёт 100%
   чистоты кластера, h_empirical = 0.0095 ровно на нижней границе
   $\lambda_1/2$

Дополняет topological_hdc.c: те же данные, две линзы
(непрерывная vs дискретная), одинаковые cluster counts.

### 3.8 clifford_hdc.c — Cl(n,0) геометрическая алгебра

**Новизна**: первый **некоммутативный** HDC binding. Использует
Cl(11, 0) алгебру Клиффорда (Clifford 1878, Hestenes 1966):

- $n = 11$, $D = 2^n = 2048$
- Мультивектор: $M = \sum_A c_A e_A$, где $A \subseteq \{0,...,10\}$,
  коэффициенты троичные $\{-1, 0, +1\}$
- **Геометрическое произведение**: $e_A \cdot e_B = \varepsilon(A,B) e_{A \oplus B}$
  где $\varepsilon(A,B) = (-1)^\sigma$, $\sigma = \sum_{a \in A} \text{popcount}(B \cap [0..a-1])$

**Шесть экспериментов**:

1. Anti-commutativity 1-vectors: 121/121 ✓
2. Non-commutativity: $\|ab - ba\|_1 / \|ab\|_1 \approx 1.33$
3. Associativity: $(ab)c = a(bc)$ на 3/3 рандомных
4. Rotor: $R \cdot e_0 \cdot \tilde R = +2 e_1$ — дискретный ротор
5. Grade decomposition: $\sum_k \text{grade}_k(M) = M$ ровно
6. **Псевдо-ортогональность сохранена**: CV = 0.039 — Clifford
   даёт некоммутативность без потери статистической надёжности HDC

### 3.9 Общие выводы фазы A

После 7 вариантов выяснилось несколько закономерностей:

1. **HDC-субстрат гибкий**: практически любая конечномерная
   алгебра переносится на D=2048 битовое представление без потери
   псевдо-ортогональности
2. **Разные операции → разные режимы**: одна и та же алгебра
   (HDC) может быть крипто-примитивом (XOR) или памятью (bundle)
3. **Метрики сходятся**: topological и spectral дают одинаковые
   cluster counts, хотя работают по-разному
4. **Некоммутативность возможна**: Clifford даёт первый HDC, где
   a⊗b ≠ b⊗a, и это совместимо с HDC-статистикой

Эта фаза создала набор инструментов, которые потом использовались
при SHA-256 анализе и при построении phase-bit иерархии.

## 4. Фаза B — SHA-256 и bit calculus

### 4.1 Поворотный момент: гипотеза пользователя

В середине фазы A пользователь предложил конкретную гипотезу:
bit-based нейросеть может **сужать поиск прообразов** для
reduced-round SHA-256. Обычный brute-force требует 2^32 вычислений
на раунд. Если сеть научится давать приор на возможные W-значения,
перебор сократится.

Это не квантовый speedup (оракул всё равно требует 2^n вычислений),
но **структурное преимущество в постобработке**: вместо
итерирования по всем входам, сеть выбирает кандидатов из обученной
памяти.

### 4.2 sha256_invert.c — 1765× speedup на R=1

**Методология**: HDV-память с N=100 000 пар (state_1, W_0).
Для каждого test state_1: top-10 ближайших по Hamming. Для каждого
кандидата W_0: локальный перебор в Hamming-шаре радиуса ≤6.
Проверка: round(IV, W_0) точно равно state_1.

**Пять экспериментов**:

1. **Hamming корреляция** между входом и выходом:
   | R | corr | |---|---|
   | 1 | +0.510 | сильная структура
   | 2 | +0.261 | эксплуатируема
   | 3 | +0.174 | слабо
   | 4 | +0.139 | почти шум
   | 5-8 | ~0 | полная лавина

2. **Avalanche** — неполная даже на R=8!
   - R=1: всего 1.5% битов меняются
   - R=8: 32.8% (не полные 50%)
   - SHA-256 диффундирует медленнее ожидаемого

3. **HDV retrieval improvement** vs random:
   - R=1: 3.66× улучшение W-Hamming
   - R=2: 1.87×
   - R=3: 1.52×
   - R=4: 1.38×

4. **ЧЕСТНАЯ end-to-end инверсия на R=1**:
   - 49/50 успехов (98%)
   - Среднее: **2.43 × 10^6** sha256_round вычислений
   - Минимум: **3398** (в 10^6 раз меньше brute force!)
   - Максимум: 9.95 × 10^6
   - Brute force: 4.29 × 10^9 (2^32)
   - **SPEEDUP: 1764.7×**

**Вывод**: гипотеза подтверждена на R=1 с конкретным ускорением
на три порядка. Но дальше не работает.

### 4.3 sha256_invert_chain.c — стена на R=2

**Попытка**: цепочечная инверсия R=2 через W_1-first retrieval.
Backward round детерминистически инвертируется, если знаем W_1;
тогда получаем state_1, на который можно применить R=1 инвертор.

**Диагностика** (оракул-тест с истинным W_1):
- 30/30 успехов с avg 1.83M вычислений — пайплайн корректен

**Диагностика HDV retrieval на W_1**:
- Ближайший state_2 в памяти 500k: Hamming 33 из 256 (нормально)
- Соответствующий W_1: Hamming 14.77 из 32
- **Random baseline**: 16.00 из 32
- **Улучшение**: всего +1.23 бита (!!)

Полная инверсия R=2: **0 из 100**.

**Вывод**: сигнал от state_2 к W_1 статистически слабый и не
локализован в Hamming-окрестности. Чистая store&retrieve стратегия
упирается в стену на R=2. Для прорыва нужен обученный проектор,
которого у нас нет.

### 4.4 bit_calculus.c — Walsh спектр

**Новизна**: дискретное исчисление на булевых функциях:

- **Дискретная производная**: $\partial_i f(x) = f(x \oplus e_i) \oplus f(x)$
- **Влияние бита**: $\text{Inf}_i(f) = \Pr[\partial_i f = 1]$
- **Walsh-Hadamard transform** (Walsh 1923): $\hat F(s) = \sum_x F(x) \cdot (-1)^{s \cdot x}$
- **Parseval**: $\sum_s \hat F(s)^2 = 2^n \cdot \sum_x F(x)^2$

**Пять экспериментов**:

1. Walsh-спектры XOR, AND, MAJ, PARITY — классические контрольные
   точки, все сходятся с теорией
2. **Parseval machine-verified** для всех функций, точность 10^{-9}
3. **Influences**: формулы через производную и через Walsh
   согласуются идеально
4. **SHA-256 Walsh-спектр**: варьируем 16 из 32 битов W[0],
   измеряем спектр каждого выходного бита state[0]

| R | max\|F̂\|/2^n | bias | I(f) | режим |
|---|---|---|---|---|
| 1 | **1.000 для bit 0** | 0.500 | 1.06 | ЛИНЕЙНО |
| 2 | 0.175 | 0.088 | 6.66 | крошится |
| 3 | 0.020 | 0.010 | **8.14** | КРИПТО |
| 4 | 0.018 | 0.009 | 7.99 | saturated |

**Поразительный факт**: bit 0 после 1 раунда **идеально линеен**
по W, потому что без переносов ниже bit 0, state[0] после round
= const ⊕ W[0]. Это открыто самим Walsh-спектром, без анализа
формул SHA.

5. Spectral distance по раундам: ||F̂||₁ скачет с 4.84 до 204 и
   насыщается

**Фазовый переход: R=3**. Линейный криптоанализ умирает полностью.

### 4.5 padic_hdc.c — ультраметрика, Cantor топология

**Новизна**: p-adic valuation $v_2(z) = $ позиция младшего
ненулевого бита. $\|z\|_2 = 2^{-v_2(z)}$. Ультраметрическое
неравенство $\|x+y\|_p \le \max(\|x\|_p, \|y\|_p)$ сильнее
треугольного.

**Четыре эксперимента**:

1. **Ультраметрика**: 100000/100000 троек ✓, 33% строгих неравенств
2. **Все треугольники isoсceles**: 100% ✓ (обязательное свойство)
3. **Every point is a ball centre**: 64 точки шара, все попарные
   расстояния ≤ радиуса, любая может быть центром — Cantor-set
   топология
4. **SHA-256 через p-adic линзу**:

| R | mean v_2 | %(v_2=0) | %(v_2<4) |
|---|---|---|---|
| 1 | 0.995 | 49.98% | 93.83% |
| 8 | 1.002 | 49.73% | 93.79% |
| теория | 1.000 | 50% | 93.75% |

Уже на R=1 SHA-256 **неотличим** от случайной функции по p-adic
метрике. Отклонения в пределах шума.

**Важный урок**: Walsh и p-adic — два разных глаза. Walsh видит
линейность (и находит её на R=1), p-adic видит статистику низших
различающихся битов (и не видит ничего). Для SHA-256 p-adic —
слишком грубая метрика, но математика ультраметрики и Cantor-set
топологии полностью рабочая.

### 4.6 bit_homology_sha.c — топологический фазовый переход

**Цель**: применить topological и spectral HDC к state-облакам
reduced-round SHA-256 и проверить, переживают ли топологические
инварианты фазовый переход, который убивает линейный криптоанализ.

**Методология**: 30 независимых датасетов по 80 state-векторов
на каждый раунд. Инварианты: max persistence, mean persistence,
Fiedler λ₁, eigengap. Сравнение с 30-run random baseline через
z-scores.

**Результат** — абсолютно чистый фазовый переход:

| R | max pers | z | Fiedler λ₁ | z |
|---|---|---|---|---|
| 1 | 0.179 | **−60.4σ** | 0.996 | **+27.6σ** |
| 2 | 0.331 | **−39.3σ** | 0.983 | **+26.5σ** |
| 3 | 0.473 | **−19.5σ** | 0.947 | **+23.5σ** |
| **4** | **0.612** | **−0.07σ** | **0.647** | **−1.0σ** |
| 6 | 0.611 | −0.31σ | 0.656 | −0.27σ |
| 8 | 0.611 | −0.24σ | 0.650 | −0.72σ |
| 16 | 0.608 | −0.65σ | 0.652 | −0.57σ |
| 64 | 0.610 | −0.42σ | 0.656 | −0.23σ |

**R=1..3**: state-облака МАССИВНО более компактные и
связанные, чем случайные (до −60σ!). **R=4**: всё резко снимается
до неотличимого от равномерного. Вплоть до полных 64 раундов.

**Mean persistence** нормализуется раньше max persistence:
- R=3: max pers −19.5σ, но mean pers уже только −7.7σ

Это значит: глобальные кластеры (max persistence) исчезают
медленнее локальных средних — последний аномальный кластер
доживает до R=3, всё остальное уже нормально.

### 4.7 Сводка фазы B: четыре независимых измерения

После фазы B у нас четыре независимых метода, все указывают на
один и тот же фазовый переход R=3→R=4:

| метод | файл | стадия «случайности» |
|---|---|---|
| Hamming correlation | sha256_invert.c | ≈5 |
| HDV retrieval collapse | sha256_invert.c | 3 |
| Walsh max\|F̂\|, I(f) | bit_calculus.c | 3 |
| **Persistent homology + Fiedler** | bit_homology_sha.c | **4** |

Это сильное совпадение. SHA-256 после 4 раундов достигает
единого состояния «максимальной случайности» сразу по четырём
ортогональным метрикам.

Одновременно стало ясно: **пробивать SHA-256 с нашими
инструментами не получится**. R=1 — игрушка. R=2 — стена. R=3+
— криптография. Пользователь скорректировал цель: вернуться к
фундаментальной математике битов и искать новые примитивы, а не
пытаться прорваться в реальную криптосистему.

## 5. Фаза C — Иерархия фазовых битов

### 5.1 Основная идея

После возврата к фундаментальной математике возник вопрос: какие
свойства есть у кубитов, которые нам хотелось бы воспроизвести
классически? Из квантовых у кубитов есть три главных дара:

1. **Фаза** — знак ±1 у амплитуды
2. **Интерференция** — деструктивная и конструктивная
3. **Запутанность** — нелокальные корреляции

Первые два можно получить **чисто классически**, если от обычного
бита перейти к **phase bit** — элементу {−1, 0, +1}, где 0 означает
«нет здесь ничего», +1 и −1 — единица с фазой. Третье (запутанность
как физический феномен) требует настоящего квантового устройства,
но **алгебраическая структура запутанности** (неразделимые пары,
Bell states, GHZ) полностью воспроизводится как phase-HDV над
{−1, 0, +1}⁴ и {−1, 0, +1}⁸.

Фаза C строит полную иерархию:
bit → phase bit → ebit → ghz → gates → limits → universal → HDC → algo.

### 5.2 phase_bits.c — первый уровень

**Примитив**: phase bit ∈ {−1, 0, +1}.

**Операции**:

- `bind(a, b) = a · b` (XOR с знаком, 0 поглощает)
- `bundle(a, b) = a + b` (целочисленное сложение — интерференция
  автоматически)
- `bundle(a, −a) = 0` ровно (**деструктивная интерференция**)

**Единственная новая способность против обычного HDC**: точное
вычитание. Бинарный XOR не имеет обратного к majority bundle — нельзя
удалить запомненный паттерн. Phase bits через минус это решают.

**Четыре эксперимента**:

1. Bind ассоциативен, a⊗a = единица, bundle(a,−a) = 0, bundle(a,a) = 2a
2. **Interference memory**: 20 паттернов, удаляем #7 через −item_7.
   cos(mem, item_7) падает с +0.22 до +0.0009, все 19 остальных
   сохранены идеально
3. **Role-unbinding search**: content-addressable память через
   bundle(items ⊗ roles), top-K по размеру N
4. **Walsh-Hadamard через phase-bit скалярное произведение**:
   $\hat F(s)$ вычисляется одним скалярным произведением —
   **WHT становится нативной операцией**

**Ключевое**: phase bits — **строгое расширение** бинарных.
Каждый бинарный HDV (без нулей и минусов) является phase HDV,
но phase HDV могут выражать минус, которого бинарные не имеют.

### 5.3 ebit_pairs.c — неразделимые пары

**Примитив**: ebit = phase-HDV длины 4, индексированный парами
(a, b) ∈ {0,1}².

**Bell states** как phase-HDV:

- Φ⁺ = (+1, 0, 0, +1) — биты совпадают, + знак
- Φ⁻ = (+1, 0, 0, −1) — биты совпадают, − знак
- Ψ⁺ = (0, +1, +1, 0) — биты противоположны, + знак
- Ψ⁻ = (0, +1, −1, 0) — биты противоположны, − знак

**Четыре эксперимента**:

1. **Ортогональный базис**: ⟨B_i | B_j⟩ = 2·δ_ij
2. **Non-factorizability** exhaustive search по коэффициентам в
   [−2,+2]: ни один Bell state не раскладывается ни в полное
   α⊗β, ни в любое бипартитное (αβ)⊗γ или α⊗(βγ)
3. **Measurement**: при измерении одного бита Bell state партнёр
   детерминирован 100% в каждой ветке. Сепарабельные состояния
   (|+⟩⊗|+⟩) показывают полную независимость партнёра
4. **Pair-relation memory**: до 64 ebit'ов в bundle длины 1024
   хранятся со 100% accuracy (стандартная HDC планка N ≤ D/16)

### 5.4 ghz_triples.c — неприводимые тройные корреляции

**Примитив**: ghz = phase-HDV длины 8, индексированный (a, b, c) ∈ {0,1}³.

**Канонические состояния**:

- GHZ⁺ = (+1, 0, 0, 0, 0, 0, 0, +1)
- GHZ⁻ = (+1, 0, 0, 0, 0, 0, 0, −1)
- W = (0, +1, +1, 0, +1, 0, 0, 0)

**Ключевой эксперимент — маргиналы**:
GHZ⁺ и GHZ⁻ имеют **идентичные** 2-битные маргиналы:
p(00) = p(11) = 1/2, корреляция +1. Знак живёт только в
трёхбитовой структуре.

**Phase-чувствительный оператор ⟨X⊗X⊗X⟩**:

| состояние | ⟨Z⊗Z⊗Z⟩ через \|amp\|² | ⟨X⊗X⊗X⟩ phase |
|---|---|---|
| GHZ⁺ | 0 | **+1** |
| GHZ⁻ | 0 | **−1** |
| W | −1 | 0 |

**Самое сильное утверждение фазы C**: вероятностное измерение
(через $|c|^2$) **не может отличить** GHZ⁺ от GHZ⁻. Phase-чувствительный
bit-flip оператор сразу видит знак. Это прямое доказательство, что
phase-bit амплитуды несут информацию, **принципиально недоступную
вероятностным распределениям**.

**Triple-relation memory**: 64 ebit'а хранятся с 100% accuracy в
bundle длины 2048 с 8 тегами на слот.

### 5.5 phase_gates.c — Deutsch-Jozsa и Bernstein-Vazirani

**Новизна**: линейно-операторный взгляд на phase HDV. Примитивные
гейты:

- **H** Hadamard: (a, b) → (a+b, a−b) (без √2 нормализации)
- **X** NOT: меняет местами половины по биту
- **Z** phase flip: инверсия знака там, где бит = 1
- **CNOT**: контроль и цель
- **WHT**: H на всех битах одновременно

**Чёткий отказ от квантового speedup**: оракул всё равно требует
2^n вычислений. Преимущество — **структурное**: одна операция WHT
после оракула коллапсирует ответ в одну амплитуду.

**Deutsch-Jozsa** (n=6, N=64), все 5 функций классифицированы
правильно:

| f | amp[\|0…0⟩] | вердикт |
|---|---|---|
| const-0 | +64 | CONSTANT |
| const-1 | −64 | CONSTANT |
| balanced x_0 | 0 | BALANCED |
| balanced parity | 0 | BALANCED |
| balanced mix | 0 | BALANCED |

**Bernstein-Vazirani** (n=8, N=256), все 5 скрытых строк
восстановлены точно: amp[0x00]=+256, amp[0xFF]=+256, amp[0xA5]=+256,
amp[0x5A]=+256, amp[0xC3]=+256.

Восстановление скрытой строки `a` из одной операции WHT после
оракула.

**Четыре алгебраических тождества** подтверждены на случайных
состояниях:

- HXH = 2·Z (с ненормализованным H)
- HZH = 2·X
- WHT² = 2^n · I
- CNOT² = I

### 5.6 phase_limits.c — no-cloning, monogamy, complementarity

**Цель**: показать, что phase bits **подчиняются тем же no-go
теоремам**, что и кубиты, хотя никакой физики нет.

**Четыре границы**:

1. **No-cloning**: CNOT клонер работает на базисных состояниях
   (|0⟩⊗|0⟩ → |0⟩⊗|0⟩, |1⟩⊗|0⟩ → |1⟩⊗|1⟩), но на суперпозиции
   |+⟩⊗|0⟩ = (1, 0, 1, 0) выдаёт **Bell state Φ⁺ = (1, 0, 0, 1)**,
   а не желаемое |+⟩⊗|+⟩ = (1, 1, 1, 1). Линейность + выполнение
   на базисе автоматически ломает клонирование суперпозиций.

2. **Monogamy**: в состоянии (Φ⁺)_AB ⊗ |0⟩_C:
   - ⟨Z_A · Z_B⟩ = +1.0000 (максимум)
   - ⟨Z_A · Z_C⟩ = +0.0000 (ноль)
   - ⟨Z_B · Z_C⟩ = +0.0000 (ноль)
   Bell-пара **исчерпывает** бюджет корреляции A.

3. **Complementarity**: Z-базис и X-базис — максимально
   комплементарны:
   - |0⟩ → Pr(Z=0)=1, Pr(X=0)=0.5
   - |+⟩ → Pr(Z=0)=0.5, Pr(X=0)=1
   Классический аналог Гейзенберга для амплитудных базисов.

4. **Schmidt rank**: 2-битное состояние как 2×2 матрица. Ранг 1 =
   сепарабельно, ранг 2 = запутано. Для 2 битов максимум ранга = 2.
   Все 4 Bell states имеют ранг 2.

### 5.7 phase_universal.c — замыкание группы

**Цель**: какие линейные операторы достижимы комбинациями
{H, X, Z, CNOT}?

**Результаты BFS**:

- **Single qubit {H, X, Z}**, max entry ≤ 8: **64 матрицы**,
  распределены по 16 на каждом уровне (1, 2, 4, 8) — наш
  ненормализованный H удваивает норму, группа бесконечна,
  но 16 «базовых» при max entry = 1 образуют Pauli-подобное
  семейство {±I, ±X, ±Y, ±Z, ...}.
- **{X, CNOT}** на 2 кубитах: ровно **24 = 4! перестановки**
  базисных состояний. Перестановочная мощность полностью
  приходит из {X, CNOT}.
- **3-CNOT SWAP** identity подтверждён: CNOT_AB · CNOT_BA · CNOT_AB = SWAP
- Полное замыкание {H, X, Z, CNOT} на 2 кубитах при cap=4:
  **2988 матриц**. Для сравнения квантовая 2-qubit Clifford
  group имеет порядок 11520.

### 5.8 phase_hdc.c — мутабельная ассоциативная память

**Новизна**: гибридная память, где ключ — бинарный HDV, а значение
— phase-HDV. Субстрат Kanerva HDC + phase-bit арифметика:

- Store: `bundle += bind(key, value)`
- **Remove**: `bundle -= bind(key, value)` ← новая операция
- Retrieve: `cos(unbind(bundle, key), guess)`
- **Overwrite**: remove old, store new

**Четыре эксперимента**:

1. Store/retrieve 32 пар → 100% accuracy, margin 0.1088
2. **Exact removal**: cos падает с +0.23 до +0.013, 19/19 сохранены
3. **Overwrite**: val1 исчез, val2 появился на месте
4. Capacity: 100% до N=32, 92% при N=64, разваливается к N=128

**Что бинарный HDC не может**: чистого удаления, in-place
overwrite, доказательства отсутствия.

### 5.9 phase_algo.c — 3-SAT counting и linearity test

**Эксперимент 1**: spectral signatures — каждая булева функция
даёт уникальный отпечаток в Walsh-амплитудах.

**Эксперимент 2**: **3-SAT counting через одну WHT**.
Phase-flip UNSAT-назначений, WHT, читаем amp[0]:
$amp[0] = \#SAT - \#UNSAT = 2 \cdot SAT - N$.

| m | ratio | #SAT brute | #SAT через WHT |
|---|---|---|---|
| 16 | 2.0 | 12 | 12 ✓ |
| 24 | 3.0 | 3 | 3 ✓ |
| 32 | 4.0 | 11 | 11 ✓ |
| 40 | 5.0 | 8 | 8 ✓ |

Точное SAT-counting за одну операцию (после построения оракула
за 2^n вычислений).

**Эксперимент 3**: **linearity test** через концентрацию
Walsh-спектра. Линейная функция ⇔ пик²/total = 1.

| функция | peak²/total | вердикт |
|---|---|---|
| linear (a=0xA5) | 1.0000 | LINEAR ✓ |
| nonlinear (AB+C) | 0.2500 | nonlinear ✓ |
| random mask 0x13 | 1.0000 | LINEAR ✓ |

### 5.10 Итоги фазы C

Полная phase-bit иерархия:

| # | примитив | добавляет |
|---|---|---|
| 0 | bit {0,1} | базовый XOR |
| 1 | phase bit | субтракция, интерференция, native Walsh |
| 2 | ebit (4-d) | неразделимые пары, Bell basis |
| 3 | ghz (8-d) | тройные корреляции, phase > probability |
| 4 | gates | линейные операторы, DJ / BV circuits |

Каждый уровень — строгое расширение предыдущего, всё на
целочисленной арифметике.

**Главное утверждение**: phase-bit амплитуды несут информацию,
принципиально недоступную вероятностным моделям. Никакое
классическое вероятностное распределение над {0,1}^3 не может
воспроизвести $\langle X \otimes X \otimes X \rangle$; наши
целочисленные phase-HDV могут.

## 6. Фаза D — Альтернативные оси примитива

### 6.1 Возвращение к исходной цели

После phase-bit иерархии возник естественный вопрос: **phase
bits — это единственный путь расширить бит, или существуют
принципиально другие оси?**

Phase, ebit, ghz, gates — все связаны одной идеей амплитуды.
Что если расширять бит **иначе**?

Четыре предложенных направления:

- **A**: Reversible bits — сохранение информации (Landauer, Bennett, Fredkin)
- **B**: Stream bits — бит как функция времени (cellular automata)
- **C**: Probabilistic bits — Bayesian распределения (Shannon)
- **D**: Braid bits — топологические инварианты (Artin, braid group)

Все четыре независимы от phase-bit ветки и друг от друга
(как потом подтвердит dependency graph).

### 6.2 reversible_bits.c — Toffoli, Fredkin

**Фундамент**: Bennett 1973, Fredkin-Toffoli 1982, Landauer 1961.
Каждый гейт — перестановка, каждая схема обратима, ни один бит
никогда не стирается, нулевая термодинамическая диссипация.

**Примитивные гейты** (все self-inverse):

- NOT: x → ¬x
- CNOT(c, t): t → t ⊕ c
- TOFFOLI(c1, c2, t): t → t ⊕ (c1 ∧ c2)
- FREDKIN(c, a, b): если c=1, меняет местами a и b

**Пять экспериментов**:

1. **Self-invertibility**: 16/16 тестов для каждого из 4 гейтов —
   каждый гейт свой собственный обратный
2. **Fredkin conservation**: 256/256 — Hamming вес сохраняется;
   TOFFOLI обратим, но НЕ консервативен (меняет вес в 64/256
   случаях, сбалансировано ±1)
3. **AND/OR через Toffoli + ancilla**: AND = 1 Toffoli + 1 ancilla,
   OR = 1 Toffoli + 1 ancilla + 5 NOTs через De Morgan
4. **Reversible full adder**: 3 CNOT + 2 Toffoli + 2 ancilla.
   Все 8 входов правильны, **обратный проход восстанавливает
   входы точно**
5. **Bennett uncomputation**: compute, copy, uncompute. Все 4
   тестовых случая — ancilla чистая

**Новое свойство против обычного бита**: **INFORMATION
CONSERVATION как жёсткое ограничение**. Ни один бит никогда
не стирается. Landauer-стоимость кТ·ln2 не платится.

### 6.3 stream_bits.c — бит как функция времени

**Примитив**: stream bit = последовательность x[0], x[1], x[2], ...
∈ {0, 1}^ω, конечный префикс длины L. Алгебра:

- `shift_k(x)[t] = x[t−k]` — временная задержка
- `XOR(x, y)` — точечный
- **Сдвиг + XOR образуют F₂-модуль**: любой shift-invariant
  линейный оператор = полином от S

**Четыре эксперимента**:

1. **LFSR** (n=7, примитивный полином x^7 + x^6 + 1):
   измеренный период = **127 = 2^7 − 1** (максимальная длина) ✓,
   в одном периоде 63 нуля и 64 единицы
2. **Shift-XOR алгебра**: S(x⊕y) = Sx ⊕ Sy (0 mismatches),
   S²x = S(Sx) ✓
3. **Cellular automata** (rule 30 и rule 110, случайные начальные
   условия): обе дают aperiodic потоки
4. **Автокорреляционные спектры** (с удалением смещения):

| τ | rule30 | rule110 | random |
|---|---|---|---|
| 0 | +1.000 | +1.000 | +1.000 |
| 1 | +0.544 | −0.028 | +0.006 |
| 2 | +0.498 | −0.112 | −0.032 |
| **7** | **+0.522** | **+0.915** | −0.008 |
| 10 | +0.401 | −0.127 | +0.021 |

- **Random**: плоский на всех τ > 0 (шум ±0.03)
- **Rule 30**: держится ~+0.5 на всех τ — хаотично, но
  структурно запоминает
- **Rule 110**: почти плоский, **КРОМЕ τ=7**, где скачок
  +0.915. Это **период глайдера** Тьюринг-полного правила 110

**Новое свойство**: **ВРЕМЯ**. Rule 110 — одно правило на одном
бите — Тьюринг-полное. Обычному биту для универсальности нужен
полный компьютер (память + процессор + управление). Здесь весь
вычислитель сводится к локальной функции обновления.

### 6.4 prob_bits.c — Bayesian распределения

**Примитив**: pbit = (p₀, p₁), p_i ≥ 0, p₀ + p₁ = 1.

**Пять экспериментов**:

1. **Joint и marginal**: outer product, трассировка по оси
   восстанавливает оригиналы; non-product joint (0.4, 0.1, 0.1, 0.4)
   имеет det = 0.15 ≠ 0, т.е. **коррелирован, не факторизуется**
2. **Bayes update** на медицинском тесте:
   P(sick) = 0.1, TPR = 0.9, FPR = 0.1 → P(sick | test+) = 0.5
   Базовая ставка 0.10 становится 0.5 — знаменитая base-rate
   fallacy как строка кода
3. **Entropy**: (1.0, 0) → 0, (0.9, 0.1) → 0.469, (0.5, 0.5) → 1.000
4. **BSC capacity**:

| f | H(f) | capacity | MI (numerical) |
|---|---|---|---|
| 0.00 | 0.0000 | 1.0000 | 1.0000 |
| 0.10 | 0.4690 | 0.5310 | 0.5310 |
| 0.30 | 0.8813 | 0.1187 | 0.1187 |
| 0.50 | 1.0000 | 0.0000 | 0.0000 |

Capacity и mutual information совпадают точно — **Шеннон
численно подтверждён**.

5. **Phase vs prob**: Bell states Φ⁺ и Φ⁻ имеют **идентичные**
   вероятностные распределения (|amp|²), различаются только знаком.
   Никакая pbit-переменная не может их различить.

**Вывод**: pbits — **строго слабее** phase bits. Не-отрицательные
амплитуды запрещают Bell-подобные знаковые структуры.

### 6.5 braid_bits.c — топологическая ось

**Фундамент**: Artin 1925. Группа B_n на n прядях имеет
генераторы σ_1, ..., σ_{n−1}:

- σ_i: прядь i идёт НАД прядью i+1
- σ_i^{−1}: прядь i идёт ПОД прядью i+1

**Отношения**:

- σ_i σ_j = σ_j σ_i при |i−j| ≥ 2 (коммутативность дальних)
- σ_i σ_{i+1} σ_i = σ_{i+1} σ_i σ_{i+1} (**Yang-Baxter**)

**Пять экспериментов**:

1. **Artin relations как перестановки**: σ_1σ_3 = σ_3σ_1 ✓,
   σ_1σ_2σ_1 = σ_2σ_1σ_2 ✓ (совпадают как перестановки)
2. **Underlying permutations**: σ_1σ_1 даёт identity как
   перестановку, но writhe = 2 — топологически нетривиально
3. **Linking number**: для σ_1σ_1 (Hopf): link(0,1) = 2;
   σ_1⁻¹σ_1⁻¹: −2; σ_1σ_1⁻¹: 0
4. **Writhe**: тривиальный = 0, Hopf = 2, trefoil = 3, figure-8 = 0
5. **Equivalence**: a = σ_1σ_2σ_1 и b = σ_2σ_1σ_2 имеют те же
   перестановки и те же writhes → Yang-Baxter на обоих инвариантах

### 6.6 braid_jones.c — Burau представление

**Углубление**: reduced Burau representation
$\rho: B_3 \to GL_2(\mathbb{Z}[t, t^{-1}])$

$$\rho(\sigma_1) = \begin{pmatrix} -t & 1 \\ 0 & 1 \end{pmatrix}, \quad
\rho(\sigma_2) = \begin{pmatrix} 1 & 0 \\ t & -t \end{pmatrix}$$

Реализовано через целочисленную арифметику Laurent-полиномов
(степени −20..+20, массив коэффициентов) и 2×2 матричные операции.

**Четыре эксперимента**:

1. Generators verified; σ_i · σ_i^{−1} = I
2. **Yang-Baxter как матричное тождество**:
   $$\rho(\sigma_1\sigma_2\sigma_1) = \rho(\sigma_2\sigma_1\sigma_2) = \begin{pmatrix} 0 & -t \\ -t^2 & 0 \end{pmatrix}$$
   Байт-идентичные матрицы. Burau представляет именно
   **braid group**, а не только симметрическую.
3. Четыре braid класса дают четыре различных матрицы:
   - тривиальный: I
   - Hopf σ₁²: [[t², 1−t], [0, 1]]
   - **trefoil σ₁³: [[−t³, t² − t + 1], [0, 1]]** — элемент
     правый-верх это **точный полином Александера трефоила 3_1**!
   - figure-8: полная Laurent-матрица с t^{-2}..t²

   Классический узловой инвариант получен прямым вычислением
   из нашей integer-polynomial реализации.
4. **Trace conjugation invariant**: tr ρ(σ₁σ₁σ₂) = 0, остаётся 0
   при сопряжении σ₂ и σ₁

### 6.7 braid_hdc.c — non-commutative HDC binding

**Идея**: использовать braid-операцию вместо XOR как binding в HDC.
Каждый braid-HDV = (v, B), где v — бинарный вектор, B — Burau-тег.

- bind(a, b): v_a ⊙ v_b (XOR), B_a · ρ(σ_1) · B_b
- tag зависит от данных через хеш HDV

**Четыре эксперимента**:

1. **Non-commutative**: v-части совпадают (XOR коммутативен),
   но Burau-теги различаются
2. **Associative** (унаследовано от матричной алгебры):
   (a⊙b)⊙c = a⊙(b⊙c) — парентезизация не важна, важен только
   линейный порядок
3. **Sequence encoding**: ABC, CBA, ACB — три различных Burau-тега
4. **Retrieval by tag**: query CBA в store {ABC, CBA, ACB, BAD, DAB, BCD}:
   - v-only match: 3/6 (неоднозначно — ABC, CBA, ACB совпадают)
   - **tag match: 1/6 (только CBA)**

Order-sensitive HDC без multivector-blowup Clifford'а.

### 6.8 anyonic_phases.c — классические "фазы" из Burau

**Идея**: в анионной физике обмен частиц может дать любую фазу
e^{iθ}, не только ±1. Классический аналог — **Burau trace как
инвариант сопряжённости**.

**Четыре эксперимента**:

1. Фазы σ₁^n для n=1..5:
   - σ₁: tr = −t + 1
   - σ₁²: tr = t² + 1
   - σ₁³: tr = −t³ + 1
   - σ₁^n: tr = (−t)^n + 1
2. **Yang-Baxter сохраняет фазу**: tr ρ(σ₁σ₂σ₁) = tr ρ(σ₂σ₁σ₂) = 0
3. **Full twist (σ₁σ₂)³ — центр B_3**:
   $$\rho((\sigma_1\sigma_2)^3) = \begin{pmatrix} t^3 & 0 \\ 0 & t^3 \end{pmatrix} = t^3 \cdot I$$
   **Скалярная матрица!** Классический факт: центр неприводимого
   представления действует скаляром. Мы получили это прямым
   вычислением.
4. Integer witnesses при t=−1: trivial=2, (σ₁σ₂)³=−2, figure-8=7

### 6.9 Первый capstone: unified_hierarchy.c (НЕВЕРНО)

После шести альтернативных осей (бинарный, phase, ebit/ghz, prob,
reversible, stream, braid) был написан первый capstone
unified_hierarchy.c с **гипотезой**: «периодическая таблица битов
содержит шесть элементов».

Четыре канонических теста:

| примитив | T1 distinguish | T2 non-commute | T3 remove | T4 interfere |
|---|---|---|---|---|
| binary | ✓ | — | — | — |
| phase | ✓ | — | ✓ | ✓ |
| probability | ✓ | — | — | — |
| reversible | ✓ | ✓ | ✓* | — |
| stream | ✓ | — | ✓ | ✓ |
| braid | ✓ | ✓ | ✓ | ✓ |

И заявление: «braid — единственный примитив, проходящий все 4
теста; шесть осей — полная таблица».

**Это заявление оказалось ложным** — позже было опровергнуто

## 7. Фаза E — Новые оси после опровержения

### 7.1 Методологический поворот

Пользователь сформулировал задачу прямо: «найди то, что я не
учёл». Вместо коллекционирования комбинаций существующих осей —
поиск **фундаментально новой оси**, не выразимой через шесть.

Для каждого кандидата надо было ответить на три вопроса:

1. Какое свойство он добавляет против всех предыдущих?
2. Какой конкретный эксперимент это доказывает?
3. Не является ли он маскировкой чего-то, что уже есть?

В этой фазе добавились пять новых осей. Каждая подтверждена
конкретным файлом с несколькими экспериментами и строгой
привязкой к стандартной математике.

### 7.2 linear_bits.c — седьмая ось: РЕСУРС

**Фундамент**: линейная логика Жирара (Girard 1987).

**Проверка ортогональности**: ни одна из шести осей не имеет
понятия «сколько раз можно прочитать бит». Binary / phase / prob
позволяют бесконечные чтения. Reversible сохраняет информацию, но
каждый бит всё равно читается сколько угодно раз. Stream
существует во всех временах. Braid — топологический объект, не
потребляемый. У всех шести — **бесплатные чтения**.

**Примитив**: linear bit = `{value, budget, consumed}`.

**Правила**:

- `read` декрементирует budget, ошибка при 0
- `drop` отбрасывает бит, забирая одно использование (нет
  бесплатного weakening)
- `clone` — ЯВНЫЙ гейт: читает source один раз, создаёт новый
  бит с указанным budget (нет бесплатной contraction)
- `!A` — бит с неограниченным budget = обычный классический бит

**Пять экспериментов** (все чистые):

1. **Single-use**: budget-1 бит, читаем дважды → первое OK,
   второе ошибка
2. **!A embedding**: budget-∞ бит читается 5/5 раз без ошибки.
   Обычные биты — это образ !A в линейной логике
3. **CLONE с зарядом**: source budget=3, клонируем в copy
   budget=2, читаем в каждом по 2 раза = 5 чтений всего
   (включая clone). Баланс верный, exhausted source выдаёт
   ошибку при следующей попытке.
4. **Linear tensor A ⊗ B**: AND, которое потребляет оба
   операнда. Оба показывают `consumed=YES` после AND.
5. **Resource conservation**: 3 бита с общим budget 6,
   один clone (заряд 1), все оставшиеся чтения учтены.
   `reads + drops = initial budget + clone charges`, нет
   нарушений.

**Связь с квантовым no-cloning**: квантовая механика запрещает
клонирование, потому что линейность гильбертова пространства
ломает клонирование суперпозиции. Линейная логика запрещает
клонирование **по дисциплине типов**. Наш классический ресурсный
учёт даёт то же no-cloning **через счётчики**, без физики.

**Новая ось: RESOURCE AWARENESS**.

### 7.3 selfref_bits.c — восьмая ось: РЕКУРСИЯ

**Фундамент**: теорема рекурсии Клини (1938), Гёдель 1931,
Тарский (неопределимость истины).

**Примитив**: self-referential bit = решение уравнения `b = f(b)`
для данной булевой функции f.

**Четыре эксперимента**:

1. **Классификация всех 4 single-bit функций**:

| функция | f(0) f(1) | фикс. точки |
|---|---|---|
| identity | 0 1 | {0, 1} — две |
| **negation** | **1 0** | **НЕТ — парадокс лжеца** |
| const 0 | 0 0 | {0} |
| const 1 | 1 1 | {1} |

Ровно одна из 4 функций (¬) не имеет фикс. точки. Это
классический лжец.

2. **Coupled 2-bit fixed points**:
   - b₀ = b₀∧b₁, b₁ = b₀∨b₁ → **3 фикс. точки**
   - NAND / XOR → 0
   - ¬b₁ / ¬b₀ → 0 (анти-коррелированный лжец)
   - implication / and → 1

Количество фикс. точек — **внутренний инвариант** coupled системы.

3. **Kleene recursion witness**: для каждой тотальной функции F
   существует b с F(b) = b. Конкретно для F_c(b) = c ⊕ b:
   - F_0: фикс. точки {0, 1}
   - F_1: нет фикс. точек

   **2-бит quine под SWAP**: фикс. точки F(a,b) = (b,a) —
   только (0,0) и (1,1). Минимальные самоописывающие паттерны.

4. **Лжец как no-fixed-point сертификат**: b = ¬b проверен
   механически, обе попытки дают противоречие. Тарский:
   истина не определима в том же языке — здесь превращено в
   **разрешимый структурный факт**.

**Ортогональность**: ни одна из семи предыдущих осей не имеет
примитивного понятия фиксированной точки. Phase, prob, reversible,
stream, braid, linear — все берут значения в фиксированном
примитивном типе. Self-ref делает решение уравнения b = f(b)
самой основой примитива.

**Новая ось: RECURSION / FIXED POINT**.

### 7.4 church_bits.c — девятая ось: ПОВЕДЕНИЕ

**Фундамент**: Church 1936, Church encoding булевых значений.

**Примитив**: Church bit — это **чистая функция**, не
хранимое значение:

- `TRUE = λxy.x` — выбери первую альтернативу
- `FALSE = λxy.y` — выбери вторую

**Никакого поля данных нет**. Бит — это функция.

**Логические операции как лямбда-термы**:

- `NOT p = λxy. p y x`
- `AND p q = λxy. p (q x y) y`
- `OR p q = λxy. p x (q x y)`
- `IF c t e = c t e` (просто применение)

В C реализация через enum + рекурсивный evaluator, но семантика
идентична Church'у.

**Четыре эксперимента** (все чистые):

1. **TRUE/FALSE как селекторы**: TRUE(7, 42) = 7, FALSE(7, 42) = 42.
   Декодирование — просто probe с (1, 0).
2. **Полные таблицы истинности NOT/AND/OR/XOR** через чистые
   Church-термы, без логики целых чисел
3. **IF это только применение**: IF TRUE then X else Y = X,
   вложенность работает без отдельной control-flow
4. **Functional extensionality**:
   - NOT(NOT TRUE) ≡ TRUE ✓
   - De Morgan: NOT(T ∧ F) = NOT T ∨ NOT F ✓
   - Absorption: T ∧ (T ∨ F) = T ✓

   Равенства доказаны пробой на всех парах входов, а не сравнением
   значений.

**Ортогональность**: ни одна из 8 предыдущих осей не ставит
функцию на уровень примитива. Во всех восьми бит — это (или
содержит) значение; здесь бит — это **вычисление, выбирающее
между x и y**.

**Новая ось: HIGHER-ORDER / FUNCTIONAL**.

### 7.5 modal_bits.c — десятая ось: KRIPKE МИРЫ

**Фундамент**: Saul Kripke, possible-worlds semantics (1963).

**Примитив**: modal bit = функция $b: W \to \{0, 1\}$ на множестве
возможных миров W, плюс отношение доступности $R \subseteq W \times W$.

**Модальные операторы**:

- $(\Box b)(w) = 1$ iff $b(w') = 1$ для всех $w'$ с $w R w'$
- $(\Diamond b)(w) = 1$ iff $b(w') = 1$ для некоторого $w'$ с $w R w'$

**Пять экспериментов**:

1. **S5 таутологии** на 4-world frame (R = полное отношение):
   p = (1,0,1,1) → □p = (0,0,0,0), ◇p = (1,1,1,1). В S5 обе
   модальности коллапсируют к константам.

2. **Duality □p = ¬◇¬p** на ring frame, 5/5 миров совпадают

3. **K-axiom** □(p→q) → (□p → □q) валиден в каждом мире на
   линейном frame. K — минимальная модальная логика.

4. **4-axiom** □p → □□p:
   - Валиден на транзитивном ≤ frame (4/4)
   - **Не валиден** на нетранзитивной цепи 0→1→2→3 (1 из 4)
   - 4-axiom точно характеризует транзитивность

5. **Модальное ≠ вероятностное**:
   Одинаковый truth vector p = (0,1,1,1), разные frames:
   - S5: □p = (0,0,0,0)
   - linear: □p = (0,1,1,1)

   Различные результаты из одинаковых truth-значений — frame
   несёт информацию, которую никакая marginal probability не
   может представить.

**Ортогональность**: ни одна из 9 предыдущих осей не имеет понятия
«в каком мире я нахожусь». Accessibility relation R — чисто
модальная структура. □ и ◇ — новые фундаментальные примитивы.

**Новая ось: MODALITY / KRIPKE FRAMES**.

### 7.6 relational_bits.c — одиннадцатая ось: СВЯЗЬ

**Фундамент**: relational algebra (Tarski 1941, Codd 1970).

**Примитив**: relational bit = подмножество $R \subseteq A \times B$,
т.е. бинарное отношение. Каждый бит говорит «связан ли a с b».

Все 10 предыдущих примитивов — **свойства одной сущности**.
Relational bit — **связь между двумя**.

**Операции** (relational algebra):

- union, intersection, converse (конечномерный случай через bitmap)
- **Composition**: $(R;S)(i,k) = \bigvee_j R(i,j) \wedge S(j,k)$
- **Transitive closure** $R^+ = R \cup R^2 \cup R^3 \cup \ldots$
- **Kleene star** $R^* = R^+ \cup \text{id}$

**Пять экспериментов** на 5-узловых графах:

1. **Union, intersection, converse** на циклах и пропускающих
   циклах — все разумные результаты
2. **Composition ассоциативна**: (a;a);a = a;(a;a) ✓
3. **Transitive closure** пути 0→1→2→3→4: R⁺ = полный
   верхний треугольник (все пары i<j достижимы)
4. **Kleene star** графа с циклом {0,1,2} и изолированным
   ребром 3→4: R* даёт полный 3×3 блок на цикловой компоненте
   плюс identity на изолированной паре
5. **Три представления одного отношения**: edge list,
   adjacency matrix, Boolean function — одна и та же
   информация, но каждое представление удобно для разных
   операций

**Ортогональность**: «коллекция битов» не имеет внутренней
топологии — это просто массив. Relation имеет **edge structure**:
composition R;S применяется к (a,c), если есть промежуточный b.
Это принципиально другое, чем любая из 10 предыдущих осей.

**Новая ось: RELATIONALITY / CONNECTION**.

### 8.2 axis_dependencies.c — граф симуляции

**Мотивация**: у нас 12 осей, но мы не знаем, **кто кого
симулирует**. Это не косметика — это реальная структура.

**Методология**: для каждой пары (X, Y) оценивается, может ли X
симулировать Y, на трёх уровнях:

- **2 native** — операции Y прямо доступны в X без дополнительной
  конструкции
- **1 encoded** — Y выразимо внутри X через нетривиальную
  конструкцию (интерпретатор, кодирование)
- **0 none** — нет известной симуляции, или структурное
  препятствие

Матрица 12×12 заполнена вручную из литературы плюс экспериментов
из репозитория, затем вычислено транзитивное замыкание, минимальный
базис (через greedy set cover) и независимые элементы.

**Результаты**:

**Нативные включения** (уровень 2):

- binary ⊂ всё (универсальное вложение)
- phase ⊃ prob (не-отрицательные амплитуды = распределения)
- ebit ⊃ phase (single-bit ebit ≡ 2-d phase HDV)
- ebit ⊃ prob (транзитивно)
- braid ⊃ phase (Burau eval в целом t даёт фазы)
- church ⊃ relational (relation это 2-arg предикат = нативно)

**Encoded симуляции** (уровень 1):

- church → self-ref (Y-комбинатор)
- church → modal, linear, rev, stream, braid, phase, ebit,
  prob (лямбда-исчисление Тьюринг-полно)
- linear ↔ modal (!A ↔ □A через S4)
- linear ↔ rev (интерпретация budget)
- stream → всё (rule 110 Тьюринг-полно)
- modal ↔ relational (accessibility relation это relation)

**Транзитивное замыкание** — покрытие каждой оси:

| ось | native | encoded | итого |
|---|---|---|---|
| binary | 1 | 0 | 1/12 |
| phase | 3 | 0 | 3/12 |
| ebit | 4 | 0 | 4/12 |
| prob | 2 | 0 | 2/12 |
| rev | 2 | 3 | 5/12 |
| **stream** | **2** | **10** | **12/12** |
| braid | 4 | 0 | 4/12 |
| linear | 2 | 3 | 5/12 |
| selfref | 2 | 0 | 2/12 |
| **church** | **3** | **9** | **12/12** |
| modal | 2 | 3 | 5/12 |
| relational | 2 | 3 | 5/12 |

**Минимальный базис (greedy set cover)**:
**{stream}** — одна ось покрывает все 12 через encoding.

Одного Тьюринг-полного примитива достаточно в принципе.

**Независимость (нативно)**: axis X независим, если ничто кроме
него самого не симулирует X нативно (кроме бинарного baseline).

**8 нативно независимых осей**:

- ebit, rev, stream, braid, linear, selfref, church, modal

**3 нативно зависимые**:

- phase ⊂ ebit, braid
- prob ⊂ phase, ebit, braid
- relational ⊂ church

### 8.3 Интерпретация графа

Транзитивное замыкание показывает, что **два примитива**
(stream и church) являются **универсальными по Тьюрингу**:
они покрывают все 12 осей через encoding.

**Но это не означает, что остальные избыточны**. Предупреждение,
которое надо явно проговорить:

Тьюринг-эквивалентность — **слабое** понятие. Stream и church
могут симулировать phase bits, но **только через интерпретатор**.
Алгебраическая структура ebit, braid, reversible и т.д. **не
сохраняется** такими кодированиями. Можно симулировать phase bits
внутри rule 110 CA, но вы не восстанавливаете Walsh-Hadamard
как нативную операцию — вы прогоняете интерпретатор.

Правильная картина поэтому:

- **Не более 8 нативных независимых осей**
- **Два «универсальных» примитива** (stream, church) покрывают
  всё через encoding, но это кросс-парадигмальная симуляция, не
  нативное вложение
- **Остальные оси — различные алгебраические миры**; граф
  симуляции записывает, кто из них может быть реализован прямо
  внутри кого

Аналогия: Turing machines могут симулировать квантовые схемы
(медленно), но мы всё равно изучаем нативные квантовые примитивы,
потому что их структура другая.

### 8.4 Статус программы после фазы F

Финальное состояние после трёх capstone-файлов:

1. **unified_hierarchy.c** — первый, ошибочный (6 осей)
2. **hierarchy_v2.c** — исправленный (12 осей), witnesses 12/12
3. **axis_dependencies.c** — граф симуляций, 8 нативно
   независимых, минимальный базис {stream} (в слабом смысле
   Тьюринг-эквивалентности)

Программа достигла естественного плато. Двенадцать осей —
достоверная оценка снизу, верхняя граница неизвестна.

## 9. Ключевые открытия и открытые вопросы

### 9.1 Положительные результаты

**P1. Фазовый переход SHA-256 на R=3→R=4 подтверждён четырьмя
независимыми методами**

Четыре полностью ортогональных инструмента (Hamming correlation,
HDV retrieval, Walsh spectrum, persistent homology + Fiedler)
сходятся на одной и той же границе:

- R=1, 2, 3: exploitable structure (Walsh ranks, linear bias)
- R=4+: все четыре метода дают «неотличимо от случайного»

Magnitude сигнала на R=1-3 — огромна (до −60σ для persistent
homology). Это говорит о том, что SHA-256 **очень быстро**
достигает криптографического режима, но **не мгновенно** — первые
три раунда имеют богатую структуру, которая позволяет reduced-round
атаки типа нашего 1765× speedup на R=1.

**P2. HDV-память даёт 1765× ускорение для R=1 инверсии SHA-256**

Конкретная работающая реализация нейро-guided инверсии:

- brute force: 4.29 × 10^9 вычислений round
- HDV-guided: 2.43 × 10^6 (среднее), 3398 (минимум)
- 98% success rate

Это **единственное числовое ускорение против реальной
криптофункции** в этой программе. Для R=2 метод упирается в
стену.

**P3. HDC — унифицирующий субстрат**

Каждая из семи алгебраических HDC-вариантов (ternary, quaternary,
topological, information, fibonacci, spectral, clifford)
переносится на 2048-битный субстрат без потери псевдо-ортогональности.
HDC оказалась очень гибкой платформой: любая конечномерная
алгебра хорошо ложится на hypervector representation.

**P4. Phase bits — строгое расширение бинарных HDC через минус**

Одно единственное добавление (знак ±) даёт:

- деструктивную интерференцию (bundle(a, −a) = 0)
- точное удаление из памяти
- native Walsh-Hadamard (один скалярный продукт = один коэффициент)
- Bell-подобные неразделимые пары
- GHZ-подобные тройные корреляции
- Deutsch-Jozsa и Bernstein-Vazirani circuits

Всё это работает **на целочисленной арифметике без √2 и физики**.

**P5. GHZ± неотличимы вероятностно, но отличимы phase-чувствительно**

Самое сильное утверждение всей программы:

- $\langle Z \otimes Z \otimes Z \rangle$ через $|c|^2$:
  GHZ⁺ и GHZ⁻ оба дают 0 (вероятности одинаковы)
- $\langle X \otimes X \otimes X \rangle$ через амплитуды:
  GHZ⁺ даёт +1, GHZ⁻ даёт −1 (phase различает)

Это **прямое доказательство**, что phase-bit амплитуды несут
информацию, **принципиально недоступную вероятностным моделям**.
Не теорема, а empirical witness через конкретный эксперимент.

**P6. Двенадцать осей расширения бита, 8 из которых нативно
независимы**

Главный итог всей программы: классическая математика битов
богаче, чем казалось. Не «шесть осей», а как минимум
**двенадцать**, причём:

- 8 нативно независимых: ebit/ghz, reversible, stream, braid,
  linear, self-ref, higher-order, modal
- 3 нативно зависимых: phase ⊂ ebit, braid; prob ⊂ phase, ebit,
  braid; relational ⊂ church
- 1 baseline: binary
- 2 универсальных через Тьюринг-эквивалентность: stream, church

### 9.2 Отрицательные результаты

**N1. HDV memory не пробивает R=2 SHA-256**

Despite 3.66× improvement в retrieval quality на R=1, сигнал от
state_R к W_t на R=2 слишком размазан: +1.23 бита лучше
случайного, недостаточно для локального поиска. Стена на R=2
фундаментальна для store&retrieve стратегии.

**N2. P-adic метрика не видит структуры SHA-256**

Уже на R=1 распределение v_2 неотличимо от геометрического
(mean 1.00, %(v_2=0)=50%, теория 50%). P-adic слишком грубая
метрика для SHA-256: модульная арифметика с переносами
автоматически рандомизирует младшие биты.

**N3. Гипотеза «шесть осей — полная таблица» — ложна**

Пять разных новых осей построены после этого заявления.
Предыдущие capstone-файлы документируют ошибку как урок.

**N4. Клонирование суперпозиции невозможно даже классически**

phase_limits.c показал, что линейные операторы, клонирующие
базисные состояния, неизбежно ломают клонирование суперпозиций.
То же самое no-cloning утверждение, что в квантовой механике,
но **следует из линейности самой по себе**, без физики.

### 9.3 Открытые вопросы

**Q1. Сколько на самом деле осей?**

Мы установили снизу 12 (или 8 нативно независимых). Верхняя
граница неизвестна. Кандидаты на следующие:

- Game-theoretic bits (стратегии в играх)
- Topos / subobject-classifier (обобщение модальных)
- History-dependent (Markov-like, но как первичная структура)
- Type-theoretic (HoTT/univalence на битах)
- Constraint bits (partial assignments с глобальными условиями)
- Spatial с curvature (биты на многообразии)
- Continuous (аналоговые биты в [0,1])

**Q2. Можно ли строго доказать независимость осей?**

Сейчас «независимость» заявляется эмпирически — находим
конкретный инвариант, которого нет у других. Более сильное
утверждение: **не существует** простой конструкции, переносящей
одну ось в другую. Это требует категориально-теоретического
аппарата, которого у нас нет.

**Q3. Существует ли минимальный «базис» расширений?**

Greedy set cover даёт {stream} как минимальный базис через
encoding, но это слабо в Тьюринг-смысле. Интересен базис в
**сильном** смысле — таком, что остальные оси получаются
прямыми операциями, а не интерпретатором.

Возможно, правильный ответ — это **не линейный базис**, а
**категория** с 12+ объектами и морфизмами между ними, задающими
допустимые трансляции.

**Q4. Можно ли получить реальное вычислительное преимущество из
неквантовых расширений?**

Phase bits дают структурные преимущества (WHT в одной операции,
3-SAT counting), но не query-speedup. Можно ли построить
**конкретный real-world алгоритм**, где одна из 12 осей даёт
ускорение против обычных битов? Пока что нет, кроме R=1
SHA-256 инверсии (что слишком узко).

**Q5. SHA-256 и другие стандартные алгоритмы — фазовый переход
R=3 универсален?**

Наши четыре метода дают одинаковую границу R=3→R=4. Это
свойство SHA-256 специально или свойство любого
хорошо-спроектированного хэша? Проверка на SHA-3, Blake2, BLAKE3,
Argon2 могла бы дать универсальный закон «R раундов достаточно
для крипто-режима», где R зависит от структуры round function.

### 9.4 Методологические наблюдения

**M1. Комбинирование работает лучше изобретения**

Каждая успешная ось — это перенос известной математики на
битовый субстрат, а не изобретение с нуля. Girard, Kleene, Church,
Kripke, Tarski, Artin, Burau, Fredkin, Clifford, Fiedler,
Carlsson, Walsh, Shannon, Bennett — весь корпус современной
логики и алгебры оказался применимым.

**M2. Честность отрицательных результатов**

Четыре раза в программе мы заявляли что-то и потом опровергали:

- «HDV memory сломает SHA-256» → только R=1
- «R=2 chain через HDV memory работает» → нет
- «p-adic метрика увидит структуру» → нет
- «Шесть осей — полная таблица» → опровергнуто 5 раз

Каждый раз опровержение оставалось в репо как **сертификат
предела метода**, а не стиралось.

**M3. Измеримость инвариантов**

Все 12 осей имеют конкретные численные witnesses (hierarchy_v2.c).
Заявлять «это новое» без такого witness было бы декларативно.
Инвариант должен быть проверяем компьютером.

**M4. Плато как точка остановки**

Программа остановлена не потому, что закончились идеи (можно
строить 13-ю ось), а потому что добавление очередной оси даёт
убывающую отдачу. Честно зафиксировано, что список открытый.

### 10.9 Что НЕ сделано

Честный список того, что могло бы быть сделано, но не сделано:

- **Формальные доказательства независимости осей**: всё
  эмпирично, нет категориально-теоретических аргументов о
  неразложимости
- **Унифицированная категория**: нет одной алгебраической
  структуры, содержащей все 12 осей как частные случаи
- **Hardware-уровень**: нет транзисторных схем для phase bits
  или braid bits
- **Практические применения**: большинство примитивов
  демонстрационные, не library-уровня
- **Проверка на других хэшах**: четыре метода фазового
  перехода проверены только на SHA-256
- **Бенчмарки скорости**: нет сравнения «сколько тактов
  стоит phase-bit WHT против обычного DFT»

Эти пункты — кандидаты на продолжение программы, если кто-то
захочет вернуться.

## Конец методички

Документ построен по разделам в реальном времени в ходе сессии.
Общее количество файлов на момент написания: 12 новых C-файлов,
созданных в этой сессии, плюс ~100 из предыдущих сессий.
Общее число осей расширения бита: **минимум 12**, из них
**минимум 8 нативно независимых**.

Главный положительный результат: классическая математика битов
богаче, чем кажется на первый взгляд. Квантовые биты — только
одно из возможных направлений развития. Остальные 11+ направлений
дают реальные новые вычислительные способности, используя только
целочисленную арифметику и существующую математику.
